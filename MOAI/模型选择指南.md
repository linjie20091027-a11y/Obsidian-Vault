# 模型选择指南：从数据特征到最优模型

> 面向 Kaggle 竞赛选手的实战指南 —— 告别盲目试模型，用数据驱动的思路选择最优方案。

---

## 1. 模型选择的核心原则

### 1.1 No Free Lunch 定理：没有万能模型

No Free Lunch（NFL）定理是机器学习领域最根本的约束之一。它的核心含义是：

> **在所有可能的问题上，所有算法的平均性能是相同的。**

换言之，如果模型 A 在某个问题上比模型 B 好，那一定存在另一个问题，模型 B 比模型 A 好。这告诉我们：

1. **不存在"永远最好"的模型。** XGBoost 在很多 Kaggle 表格赛中是王者，但在图像赛里毫无用处。BERT 是 NLP 的统治级模型，但用在 500 行数据上就是灾难。
2. **模型选择必须基于数据特征。** 你必须根据数据的规模、类型、质量和目标变量来选择模型，而不是凭"这个模型最近很火"。
3. **一切以验证集得分为准。** NFL 的推论是：你的任务上哪个模型好，只有跑一遍才知道。理论分析可以缩小候选范围，但不能替代实验。

**实战经验：** 在 Kaggle 竞赛中，我见过有人用简单的 Logistic Regression 拿金牌，也见过有人用复杂的 ensemble 排名末尾。关键不在于模型有多 fancy，而在于模型和数据是否匹配。

### 1.2 偏差-方差权衡（Bias-Variance Tradeoff）详解

理解偏差-方差权衡是模型选择的理论基础。我们先把概念搞清楚：

#### 偏差（Bias）

**定义：** 模型预测值的期望与真实值之间的差距。可以理解为模型的"拟合能力"。

- **高偏差** = 模型太简单，学不到数据的复杂模式 → **欠拟合（Underfitting）**
- **表现：** 训练集误差大，验证集误差也大，两者相近
- **典型情况：** 线性回归拟合非线性数据、决策树深度太浅
- **解决思路：** 增加模型复杂度（更多特征、更深的树、网络层数）、减少正则化

#### 方差（Variance）

**定义：** 模型在不同训练集上预测结果的变化程度。可以理解为模型的"稳定性"。

- **高方差** = 模型对训练数据过于敏感，稍微换一批数据结果就大变 → **过拟合（Overfitting）**
- **表现：** 训练集误差很小，验证集误差很大，两者差距明显
- **典型情况：** 深度决策树、未正则化的高维线性回归、参数量远超数据量的神经网络
- **解决思路：** 增加正则化、更多数据、降低模型复杂度、集成方法（Bagging）

#### 数学表达

对于回归问题，期望平方误差可以分解为：

```
E[(y - f̂(x))²] = Bias² + Variance + σ²（不可约误差）
```

这意味着：**总要误差不可能为零**（σ² 是数据本身的噪声），我们能做的就是找到 Bias² 和 Variance 的最优平衡点。

#### 偏差-方差与模型复杂度的关系

```
误差
 ↑
 |    ┌──────────────────────────────────┐
 |    │  总误差  ╱╲        方差         │
 |    │        ╱    ╲      ╱            │
 |    │      ╱        ╲  ╱              │
 |    │    ╱        ╱╲╱                 │
 |    │  ╱     ╱╲╱                      │
 |    │╱   ╱╲╱   偏差²                  │
 |    └──────────────────────────────────→ 模型复杂度
         最优复杂度点
```

- **模型太简单** → 高偏差，低方差
- **模型太复杂** → 低偏差，高方差
- **最佳点** → 总误差最低

#### 不同模型的偏差-方差特征

| 模型 | 偏差 | 方差 | 说明 |
|------|------|------|------|
| 线性回归（无正则化） | 中高 | 低 | 只能拟合线性关系 |
| Ridge（L2 正则化） | 中高 | 低 | 偏差略增，方差更低 |
| Lasso（L1 正则化） | 中 | 中低 | 特征选择降方差 |
| 决策树（浅层） | 高 | 低 | 简单规则，拟合能力差 |
| 决策树（深层） | 低 | 高 | 容易过拟合每个细节 |
| 随机森林 | 低 | 中 | Bagging 有效降方差 |
| GBDT（弱正则化） | 极低 | 中高 | 拟合能力极强，需防范过拟合 |
| GBDT（强正则化） | 低 | 中 | 经典 Kaggle 配置 |
| SVM（RBF 核） | 低 | 中 | 核技巧容量大但正则化强 |
| KNN（K 小） | 低 | 高 | K 越小越容易过拟合 |
| KNN（K 大） | 高 | 低 | K 越大越平滑 |
| 神经网络（大） | 极低 | 高 | 需要大量数据和强正则化 |
| 朴素贝叶斯 | 高 | 低 | 强假设导致高偏差 |

### 1.3 模型复杂度与数据量的关系

这是一个容易被初学者忽略的重要关系：

```
模型有效复杂度 ≈ 数据量 / 特征量
```

**核心规律：**

| 数据量 | 推荐模型复杂度 | 原因 |
|--------|----------------|------|
| 极小（<500） | 极简模型 | 复杂模型方差爆炸，正则化无法弥补数据不足 |
| 小（500-5K） | 线性模型 + 强正则化 | GBDT 可能勉强可以但要限制 depth 和 num_leaves |
| 中（5K-100K） | GBDT、中等 NN | 树模型的最佳领域 |
| 大（100K-1M） | GBDT、NN | NN 开始体现优势 |
| 极大（>1M） | NN 或 LightGBM | NN 的样本效率最低但上限最高 |

**实战要诀：**

- **数据量 < 特征数的 10 倍** → 必须正则化，优先线性模型或简单树
- **数据量 > 特征数的 100 倍** → 可以放心用深度模型
- **高维小样本（如基因数据）** → Ridge 或 ElasticNet 是可靠选择

### 1.4 Occam's Razor：优先选简单模型作为 Baseline

奥卡姆剃刀原则在机器学习中的含义是：

> **在性能相近的模型中，选择更简单的那个。**

这是竞赛中被反复验证的黄金法则：

1. **Baseline 一定要简单。** 理想情况下你的第一个模型应该在 5 分钟内训练完、10 行代码搞定。如果 Logistic Regression 或简单的 Random Forest 就能给你 0.85 AUC，那你知道上限至少是 0.85，接下来再考虑用更复杂的模型 push 到 0.87。
2. **简单模型的好处：**
   - 快速迭代（验证特征工程是否有效）
   - 容易 debug（为什么会过拟合？是特征问题还是模型问题？）
   - 为复杂模型提供参考（复杂模型至少要比简单模型好才有意义）
3. **常见 Baseline 选择：**
   - 表格分类：Logistic Regression 或 浅层 Random Forest（max_depth=6）
   - 表格回归：Ridge 或 浅层 Random Forest
   - 图像：ResNet18（而非 ViT 或 ConvNeXt）
   - 文本：TF-IDF + Logistic Regression（而非直接上 BERT）
   - 时间序列：Naive forecast（last value）或 ARIMA

**一个血泪教训：**

> 某 Kaggle 竞赛中，一位选手花了 3 天搭了一个 3 层 stacking ensemble，线下 CV 0.902。后来他试了一个 XGBoost baseline，调了半小时参数，线下 CV 0.914。那个复杂 ensemble 不仅分数更低，还因为过拟合导致线上得分崩了。

## 2. 根据数据特征选择模型（核心章节）

### 2.1 数据规模维度

数据规模是模型选择的首要因素。下面按数据量和特征量两个维度给出推荐。

#### 完整矩阵

| 数据量 | 特征量（相对数据） | 推荐模型 | 不推荐 |
|--------|---------------------|----------|--------|
| <1K | 少 | 小 MLP、朴素贝叶斯、浅层决策树 | 任何深度模型 |
| <1K | 多 | **Ridge/ElasticNet（首选）**、SVM（RBF）、KNN | Lasso（不稳定）、GBDT（方差大） |
| 1K-10K | 少 | 简单 GBDT、随机森林 | NN（数据不够） |
| 1K-10K | 多 | Ridge、XGBoost（限制深度）、SVM | NN |
| 10K-100K | 任意 | **XGBoost/LightGBM/CatBoost（最佳区间）** | - |
| 100K-1M | 少-中 | LightGBM（首选，速度最快）、TabNet | - |
| 100K-1M | 多 | LightGBM、NN（MLP） | - |
| >1M | 少-中 | LightGBM | - |
| >1M | 多 | **神经网络**、LightGBM | 传统 SVM/KNN（太慢） |

#### 各数据量的实战详解

**小数据（<1K）：**

```python
# 典型场景：稀有事件预测、小样本医学数据
from sklearn.linear_model import LogisticRegression, Ridge
from sklearn.svm import SVC
from sklearn.model_selection import StratifiedKFold

# 首选方案：正则化线性模型
model = LogisticRegression(
    penalty='l2',       # Ridge 正则化
    C=0.1,              # 强正则化（C 越小正则化越强）
    solver='liblinear',
    class_weight='balanced'  # 类别不平衡时必备
)

# 小数据的 CV 策略 —— K 要大，充分利用每一条数据
cv = StratifiedKFold(n_splits=10, shuffle=True, random_state=42)

# 评估指标 —— 小数据不要只看准确率
from sklearn.metrics import roc_auc_score, f1_score, precision_recall_curve
```

**中等数据（1K-100K）：GBDT 的黄金区间**

这是 Kaggle 表格竞赛最常见的规模。GBDT（Gradient Boosting Decision Tree）在这个区间几乎无敌：
- 训练速度快（LightGBM 可以在几秒内完成）
- 不需要特征缩放
- 天然处理缺失值
- 自动特征交互
- 对异常值鲁棒

```python
import lightgbm as lgb
from sklearn.model_selection import KFold

params = {
    'objective': 'binary',
    'metric': 'auc',
    'boosting_type': 'gbdt',
    'num_leaves': 31,
    'learning_rate': 0.05,
    'feature_fraction': 0.8,
    'bagging_fraction': 0.8,
    'bagging_freq': 5,
    'verbose': -1,
    'seed': 42,
}

# 中等数据的标准 CV 配置
cv = KFold(n_splits=5, shuffle=True, random_state=42)
```

**大数据（>100K）：LightGBM 仍可能是最优解**

LightGBM 在大数据上训练速度极快，很多 Kaggle 大数据赛道的金牌方案仍然是 LightGBM ensemble。只有在数据量超过数百万、且特征以原始像素或文本 token 为主时，神经网络才成为必然选择。

```python
# 大数据场景的 LightGBM 配置要点
params_large = {
    'num_leaves': 255,           # 大数据可以更深
    'learning_rate': 0.02,       # 降低学习率，增加迭代次数
    'subsample': 0.8,
    'colsample_bytree': 0.8,
    'min_child_samples': 100,    # 增大以增加正则化
    'reg_alpha': 0.1,
    'reg_lambda': 1.0,
    'n_estimators': 10000,       # 用 early stopping 控制
}
```

**高维稀疏数据：线性模型是王者**

当特征维度远超样本数（如 TF-IDF 文本特征、One-Hot 编码的高基数类别特征），或大多数特征值为 0 时：

| 特征类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 稀疏连续特征 | Ridge | L2 正则化稳定，所有特征都有贡献 |
| 希望自动特征选择 | Lasso | L1 正则化会将不重要的特征系数压缩为 0 |
| 两者兼顾 | ElasticNet | 既有特征选择又有稳定性 |
| 文本 TF-IDF | Logistic Regression | 经典组合，速度快效果不错 |
| 文本 TF-IDF（想更好） | 轻量 GBDT | 但树模型对稀疏特征不够擅长 |

```python
# 高维稀疏场景的标准 pipeline
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline

pipeline = Pipeline([
    ('tfidf', TfidfVectorizer(
        max_features=100000,
        ngram_range=(1, 3),
        sublinear_tf=True,   # 1 + log(tf)，抑制高频词
    )),
    ('clf', LogisticRegression(
        C=1.0,
        penalty='l2',
        solver='sag',          # 大数据量用 saga/sag，比 lbfgs 快
        max_iter=1000,
    ))
])
```

### 2.2 数据类型维度

#### 2.2.1 纯表格数据：GBDT 系列是王者

在 Kaggle 的历史上，80% 以上的表格竞赛金牌方案都以 GBDT 为核心。这不是偶然，有深层原因：

**GBDT 在表格数据上的独特优势：**

1. **对特征尺度不敏感：** 决策树基于分裂点，不需要归一化或标准化。Age=35 和 Income=50000 可以直接使用。
2. **天然处理混合类型：** 连续变量（年龄）、有序类别（学历）、无序类别（城市）可以混合输入，不需要分别处理。
3. **自动捕捉特征交互：** 树结构天然实现了"if 年龄>30 and 收入>50K then..."这样的非线性交互，不需要手动构造交叉特征。
4. **对异常值鲁棒：** 决策树的分裂基于排序，极端的异常值只会影响一个叶节点。
5. **缺失值原生支持：** XGBoost 和 LightGBM 会自动学习缺失值的最优分裂方向，CatBoost 更有专门的缺失值处理策略。
6. **训练速度快：** LightGBM 直方图算法 + leaf-wise 生长，训练速度比其他方案快一个数量级。

**为什么神经网络在表格数据上不行？**

- **样本效率低：** NN 需要大量数据来学习，而表格数据通常不够大。
- **对超参数敏感：** 学习率、层数、神经元数、batch size、dropout rate……调参空间巨大。
- **损失景观不光滑：** 表格数据的损失面高噪声、非凸，NN 的 SGD 优化不稳定。
- **缺乏旋转不变性 / 平移不变性：** 表格特征的排列顺序不应该影响结果，但 NN 会受特征顺序影响（Transformer 除外，但 TabNet/FT-Transformer 也只是在某些场景下才优于 GBDT）。

**GBDT 三剑客快速对比：**

| 特性 | XGBoost | LightGBM | CatBoost |
|------|---------|----------|----------|
| 树生长策略 | Level-wise | Leaf-wise | Symmetric |
| 默认类别处理 | 需手动编码 | 支持（需标记） | **原生自动处理** |
| 缺失值处理 | 学习最优方向 | 学习最优方向 | 多种填充策略 + 学习 |
| 训练速度 | 中 | **快（最快）** | 中 |
| GPU 支持 | 好 | 好 | 好 |
| 调参难度 | 中 | 中 | **低（默认就很好）** |
| 小数据精度 | 中 | 中 | **高** |
| Kaggle 流行度 | 极高 | 极高 | 高 |

```python
# 三大框架的基本调用示例

# ==================== XGBoost ====================
import xgboost as xgb

xgb_model = xgb.XGBClassifier(
    n_estimators=1000,
    learning_rate=0.05,
    max_depth=6,
    subsample=0.8,
    colsample_bytree=0.8,
    reg_alpha=1.0,       # L1 正则化
    reg_lambda=1.0,      # L2 正则化
    tree_method='hist',  # 使用直方图算法加速
    early_stopping_rounds=100,
    eval_metric='auc',
    random_state=42,
)

# ==================== LightGBM ====================
import lightgbm as lgb

lgb_model = lgb.LGBMClassifier(
    n_estimators=1000,
    learning_rate=0.05,
    num_leaves=31,
    max_depth=-1,         # 默认不限制，由 num_leaves 控制
    subsample=0.8,
    colsample_bytree=0.8,
    reg_alpha=0.1,
    reg_lambda=1.0,
    min_child_samples=20,
    random_state=42,
    verbose=-1,
)

# ==================== CatBoost ====================
from catboost import CatBoostClassifier

cb_model = CatBoostClassifier(
    iterations=1000,
    learning_rate=0.05,
    depth=6,
    l2_leaf_reg=3.0,     # L2 正则化（CatBoost 特有参数）
    random_strength=1.0,  # 随机性控制
    bagging_temperature=0.5,
    early_stopping_rounds=100,
    eval_metric='AUC',
    random_seed=42,
    verbose=0,
)
```

#### 2.2.2 图像数据：CNN（ResNet/EfficientNet）

图像竞赛的核心思路：

1. **Backbone 选择：**
   - **ResNet** 系列：最经典的 baseline，ResNet50/101/152 是起点。不容易出大错。
   - **EfficientNet** 系列：同参数下精度更高。EfficientNet-B3/B5 在 Kaggle 图像竞赛中非常常见。
   - **ViT（Vision Transformer）**：大数据（>1M）时效果更好，需要更多数据。
   - **ConvNeXt**：ResNet 的现代化重设计，训练效率更高。

2. **实战 Pipeline：**
   ```python
   import torch
   import torchvision.models as models
   
   # 经典 ResNet50 配置
   model = models.resnet50(pretrained=True)
   in_features = model.fc.in_features
   model.fc = torch.nn.Linear(in_features, num_classes)
   
   # EfficientNet-B3（需要 timm 库）
   import timm
   model = timm.create_model('efficientnet_b3', pretrained=True, num_classes=num_classes)
   
   # 图像竞赛关键技巧
   # - 多尺度训练和测试（Multi-scale training/test）
   # - Test Time Augmentation (TTA)：水平翻转 + 多尺度 + 旋转
   # - 使用 AdamW 优化器而不是 SGD（现代方案）
   # - Cosine annealing 学习率调度
   # - Label smoothing（减少过拟合，校准置信度）
   ```

3. **选择指南：**
   | 数据量 | 推荐方案 |
   |--------|----------|
   | <5K | 小 ResNet（18/34）+ 强数据增强 + 迁移学习 |
   | 5K-50K | ResNet50/EfficientNet-B3 + 微调最后几层 |
   | 50K-500K | EfficientNet-B5/B7 + 全模型微调 |
   | >500K | ConvNeXt/ViT + 从头训练或大规模预训练 |

#### 2.2.3 文本数据：Transformer（BERT/DeBERTa）

NLP 竞赛的模型选择几乎已经被 Transformer 垄断，但仍有关键选择：

**Baseline 路径：**
1. **TF-IDF + Logistic Regression：** 永远是 NLP 竞赛的第一个 baseline。速度快、可解释性强、能快速判断数据质量。
2. **FastText / Word2Vec + LSTM/GRU：** 如果 Transformer 资源不够时的备选。
3. **BERT-base：** 标准起点。
4. **DeBERTa-v3-base/large：** 当前 Kaggle NLP 竞赛的 go-to 模型。

**实战建议：**

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification

# DeBERTa-v3 在 Kaggle 上几乎是最优选择
model_name = "microsoft/deberta-v3-base"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(
    model_name, num_labels=num_classes
)

# 关键配置
# - max_length：根据数据分布设定（90% 分位数 + 一些 buffer）
# - 梯度累积：小 batch size 时的必备技巧
# - Mixed precision（fp16）：训练速度翻倍
# - 分层学习率：底层小，顶层大
```

**LLM 时代的新范式：**
- 使用 LLM API（GPT-4, Claude）做数据增强或伪标签
- 开源 LLM（Llama, Mistral 7B）做 LoRA 微调
- Prompt Engineering + 集成

#### 2.2.4 时间序列：从简单到复杂

时间序列竞赛的模型选择层次：

| 层级 | 模型 | 适用场景 |
|------|------|----------|
| 第 1 层 | Naive forecast / 移动平均 | Baseline，判断任务难度 |
| 第 2 层 | LightGBM + lag features | 大多数表格化时间序列竞赛的银牌方案 |
| 第 3 层 | ARIMA / SARIMA | 单变量、有明显季节性和趋势 |
| 第 4 层 | LSTM / GRU | 多变量、长序列、有复杂时序依赖 |
| 第 5 层 | Transformer / Informer | 超长序列、多元预测 |
| 第 6 层 | N-BEATS / N-HiTS | 专门为时间序列设计的深度学习模型 |

**LightGBM + lag features 的实战优势：**

```python
import pandas as pd
import lightgbm as lgb

# 构造 lag features（时间序列特征化的核心）
def create_lag_features(df, target_col, lags=[1, 2, 3, 7, 14, 28]):
    """创建滞后特征和滚动统计特征"""
    for lag in lags:
        df[f'lag_{lag}'] = df.groupby('id')[target_col].shift(lag)
    
    # 滚动统计
    for window in [7, 14, 28]:
        df[f'rolling_mean_{window}'] = df.groupby('id')[target_col].transform(
            lambda x: x.shift(1).rolling(window).mean()
        )
        df[f'rolling_std_{window}'] = df.groupby('id')[target_col].transform(
            lambda x: x.shift(1).rolling(window).std()
        )
    return df

# 时间序列预测的 CV 策略：TimeSeriesSplit
from sklearn.model_selection import TimeSeriesSplit
tscv = TimeSeriesSplit(n_splits=5)
```

```python
class GCN(torch.nn.Module):
    def __init__(self, in_channels, hidden_channels, out_channels):
        super().__init__()
        self.conv1 = GCNConv(in_channels, hidden_channels)
        self.conv2 = GCNConv(hidden_channels, out_channels)
```

#### 2.3.3 回归

回归任务的模型选择路径：

```
均值/中位数预测（最简单的 baseline）
    → Ridge/Lasso（线性 baseline）
        → GBDT（XGBoost/LightGBM/CatBoost Regressor）
            → 量化回归（分位数回归，用于不确定性估计）
```

**回归的特殊注意点：**

1. **目标变量分布：** 如果目标变量严重偏斜，考虑 log1p 变换。
2. **评估指标选择：**
   - RMSE：对极端误差惩罚大（因为是平方）。对异常值敏感。
   - MAE：对异常值更鲁棒。中位数无偏预测 vs 均值无偏预测。
   - RMSLE（Root Mean Squared Logarithmic Error）：当目标跨越多个数量级时使用。
   - MAPE：适合关心相对误差的场景。
3. **预测的目标：** 如果评估指标是 RMSE，模型预测的应该是条件均值。如果评估指标是 MAE，模型预测的是条件中位数。

```python
# 回归的标准流程

# Step 2: 对偏斜目标做 log 变换

# Step 3: 训练模型
model = lgb.LGBMRegressor(
    objective='regression',  # 或 'regression_l1'（MAE 优化）
    metric='rmse',
    n_estimators=1000,
    learning_rate=0.05,
    num_leaves=31,
)

# Step 4: 预测后逆变换
pred_log = model.predict(X_test)
pred = np.expm1(pred_log)

# Step 5: 根据评估指标选择训练目标
# 如果评估指标是 RMSE，用 objective='regression'（优化 MSE）
# 如果评估指标是 MAE，用 objective='regression_l1'
```

### 2.4 数据质量维度

数据质量直接影响模型选择。**高质量的脏数据处理比选什么模型更重要**。

#### 2.4.1 大量缺失值

| 缺失比例 | 推荐模型 | 处理策略 |
|----------|----------|----------|
| <10% | 任意模型 | 中位数/均值填充 + 缺失指示特征（is_missing 列） |
| 10%-30% | 树模型优先 | 模型原生处理 + 缺失指示 |
| 30%-50% | **CatBoost 优先** | 树模型 + 缺失指示 |
| >50% | CatBoost + 极端处理 | 该特征是否可用？考虑抛弃 |

**树模型如何处理缺失值：**

树模型（XGBoost/LightGBM/CatBoost）在训练时会将缺失值分配到"最优"的子节点（使得增益最大的方向）。这实际上是一种**学习缺失值的最优填补策略**。

```python
# 缺失值处理的最佳实践
import pandas as pd
import numpy as np

def handle_missing_for_trees(df):
    """
    树模型场景下的缺失值处理
    - 保持 NaN（树模型会自行处理）
    - 添加缺失指示特征（强烈推荐！）
    """
    df_processed = df.copy()
    for col in df.columns:
        missing_pct = df[col].isnull().mean()
        if 0 < missing_pct < 0.6:  # 缺失在合理范围内
            # 添加缺失指示特征
            df_processed[f'{col}_is_missing'] = df[col].isnull().astype(int)
            # 保持 NaN 让树模型自行处理
            # 但如果缺失超过 50%，可以用特殊值填充（如 -999）
        elif missing_pct >= 0.6:
            # 缺失太多，考虑：1) 丢弃该特征  2) 仅保留 is_missing 指示列
            df_processed.drop(col, axis=1, inplace=True)
    return df_processed
```

**CatBoost 处理缺失值的特殊方式：**

CatBoost 对缺失值的处理最精细化：
1. 训练时：将缺失值作为独立的分裂候选，或学习最优填充值
2. 预测时：自动使用训练时学到的缺失处理策略
3. 不需要手动填充，也不需要删除含缺失的行

#### 2.4.2 类别不平衡

类别不平衡是 Kaggle 竞赛中最常见的数据质量问题之一。

**严重程度分级：**

| 不平衡比（负:正） | 严重程度 | 对策 |
|-------------------|----------|------|
| <10:1 | 轻微 | 正常训练，关注评估指标选择 |
| 10:1 - 100:1 | 中等 | 使用 `class_weight` 或 `scale_pos_weight` |
| 100:1 - 1000:1 | 严重 | 过采样（SMOTE/ADASYN）+ 代价敏感学习 |
| >1000:1 | 极端 | 异常检测思路 + 专门的不平衡学习框架 |

**完整策略矩阵：**

```python
# ======== 策略 1：模型内置权重（首选） ========
from lightgbm import LGBMClassifier

# 自动计算权重
model = LGBMClassifier(class_weight='balanced')

# 手动设置
pos_weight = len(y[y==0]) / len(y[y==1])  # sklearn 风格的 class_weight
model = LGBMClassifier(scale_pos_weight=pos_weight)

# ======== 策略 2：重采样 ========
from imblearn.over_sampling import SMOTE, ADASYN
from imblearn.under_sampling import RandomUnderSampler
from imblearn.combine import SMOTEENN, SMOTETomek

# SMOTE（合成少数类过采样）—— 最常用
smote = SMOTE(sampling_strategy=0.3, random_state=42)  # 过采样到 30%
X_res, y_res = smote.fit_resample(X, y)

# SMOTE + Tomek Links（过采样 + 清洗边界）
smote_tomek = SMOTETomek(sampling_strategy=0.5, random_state=42)
X_res, y_res = smote_tomek.fit_resample(X, y)

# ======== 策略 3：集成采样（每个子模型用不同的采样） ========
# BalancedBaggingClassifier / BalancedRandomForestClassifier
from imblearn.ensemble import BalancedRandomForestClassifier
model = BalancedRandomForestClassifier(
    n_estimators=100,
    sampling_strategy='auto',  # 每棵树训练时自动欠采样多数类
    replacement=True,
    random_state=42,
)

# ======== 策略 4：阈值移动（训练后用） ========
# 不改变训练过程，而是调整预测阈值
from sklearn.metrics import precision_recall_curve
precision, recall, thresholds = precision_recall_curve(y_true, y_pred_proba)
# 选择使 F1 最大的阈值
f1_scores = 2 * precision * recall / (precision + recall + 1e-10)
best_threshold = thresholds[f1_scores.argmax()]

pred_labels = (y_pred_proba >= best_threshold).astype(int)
```

**类别不平衡时的评估指标关键提醒：**

- **不要用 Accuracy！** 99% 负类时，全预测负就有 99% 准确率，但模型毫无价值。
- **优先用 AUC（ROC-AUC）：** 对类别不平衡不敏感，衡量排序能力。
- **F1 / Precision-Recall AUC：** 当正类非常重要时（如欺诈检测）。
- **MCC（Matthews Correlation Coefficient）：** 对严重不平衡更公平的指标。

#### 2.4.3 噪声大

数据噪声大（标签错误、测量误差大、离群值多）时的模型选择：

| 噪声类型 | 推荐模型 | 原因 |
|----------|----------|------|
| 标签噪声 | **集成模型（Bagging 优先）** | 多个模型取平均/投票，降低个别错误标签的影响 |
| 特征噪声 | Ridge + 强正则化 | L2 正则化将系数收缩，不被噪声特征带偏 |
| 离群值（outliers） | GBDT | 树模型基于排序，单个极端值不主导分裂 |
| 目标噪声大 | 分位数回归 | 预测分位数而非均值，天然鲁棒 |

```python
# 标签噪声场景：使用 Bagging（随机森林）
from sklearn.ensemble import RandomForestClassifier
# 多个模型投票，个别错误标签的影响被"稀释"
rf = RandomForestClassifier(
    n_estimators=500,    # 更多树，更强的平滑效果
    max_depth=8,         # 限制深度，防止拟合噪声
    min_samples_leaf=10, # 每片叶子至少 10 个样本
    random_state=42,
)

# 离群值场景：XGBoost 中增大 reg_lambda 和 min_child_weight
xgb_model = xgb.XGBRegressor(
    n_estimators=500,
    learning_rate=0.05,
    max_depth=4,           # 浅树，更鲁棒
    min_child_weight=10,   # 叶片最小样本权重（增加以降低离群值影响）
    reg_lambda=5.0,        # 强 L2 正则化
    reg_alpha=1.0,         # L1 正则化
)

# 特征噪声场景：特征层面的降噪
from sklearn.decomposition import PCA
from sklearn.preprocessing import RobustScaler  # 对离群值鲁棒的标准化

# RobustScaler 使用中位数和 IQR，而不是均值和标准差
scaler = RobustScaler()
X_scaled = scaler.fit_transform(X)

# PCA 降噪（只保留主成分，丢弃噪声维度）
pca = PCA(n_components=0.95)  # 保留 95% 方差
X_denoised = pca.fit_transform(X_scaled)
```

#### 2.4.4 特征相关性高

当特征之间高度相关（多重共线性）时：

**为什么不能用 Lasso？** Lasso 在高度相关特征中会随机选择其中一个，而被选中的特征随数据扰动可能完全不同 → **Lasso 在高相关特征下不稳定**。

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 特征高度相关 + 需要正则化 | **Ridge** | L2 正则化会将相关特征的系数平方和分散，稳定 |
| 既要特征选择又要稳定 | **ElasticNet** | L1+L2 混合正则化，取两者之长 |
| 特征高度相关 + 需要降维 | **PCA + 模型** | 将相关特征合并为主成分，消除共线性 |
| 特征高度相关 + 用树模型 | 直接使用 | 树模型天然不受共线性影响 |

```python
from sklearn.linear_model import Ridge, ElasticNet, Lasso
from sklearn.decomposition import PCA
from sklearn.pipeline import make_pipeline

# 方案 1：Ridge —— 最稳定
ridge = Ridge(alpha=1.0)

# 方案 2：ElasticNet —— 平衡选择
elastic = ElasticNet(
    alpha=1.0,
    l1_ratio=0.5,     # L1 和 L2 各占一半
    max_iter=10000,
)

# 方案 3：PCA + Ridge Pipeline
pipeline = make_pipeline(
    PCA(n_components=0.99),  # 先降维
    Ridge(alpha=1.0),
)

# 方案 4：直接用 GBDT（树模型不受相关性影响）
# 但注意：高相关特征对特征重要性分析有干扰（重要性会被稀释）
import lightgbm as lgb
model = lgb.LGBMClassifier(n_estimators=1000, learning_rate=0.05)
```

### 3.1 线性模型家族

线性模型是机器学习的基础，也是最重要的 baseline。

#### 3.1.1 Linear Regression / Logistic Regression

**核心公式：**

- **线性回归：** `y = w·x + b`，最小化 `MSE = (1/n) Σ (y_true - y_pred)²`
- **逻辑回归：** `P(y=1) = σ(w·x + b)`，最小化交叉熵损失

**优点：**
- 训练极快（几毫秒到几秒）
- 高度可解释（系数直接表示特征影响力）
- 不容易过拟合（低方差）
- 支持稀疏特征（通过稀疏矩阵存储）

**缺点：**
- 只能拟合线性关系
- 对特征缩放敏感
- 对离群值敏感
- 需要手动特征交互

**超参数要点：**

```python
from sklearn.linear_model import LogisticRegression

# 关键参数
model = LogisticRegression(
    C=1.0,                # 正则化强度的倒数，越小正则化越强
    penalty='l2',         # 'l1'（Lasso）、'l2'（Ridge）、'elasticnet'
    solver='lbfgs',       # 小数据用 'liblinear'，大数据用 'saga'
    class_weight='balanced',
    max_iter=1000,        # 大数据时可能需要更多
    tol=1e-4,             # 收敛容忍度
)
```

#### 3.1.2 Ridge（L2 正则化）vs Lasso（L1 正则化）vs ElasticNet

| 特性 | Ridge | Lasso | ElasticNet |
|------|-------|-------|------------|
| 正则化项 | `α Σw²` | `α Σ|w|` | `α(ρ Σ|w| + (1-ρ) Σw²)` |
| 系数效果 | 缩小但不为零 | 部分压缩为 0（特征选择） | 兼顾两者 |
| 特征选择 | ❌ | ✅ | ✅（部分） |
| 高相关特征 | 稳定 | 不稳定（随机选一个） | 相对稳定 |
| 适用场景 | 特征都有用 | 需要自动选特征 | 特征多且相关 |
| 解的稀疏性 | 不稀疏 | 稀疏 | 部分稀疏 |
| 调参重点 | alpha | alpha | alpha + l1_ratio |

```python
from sklearn.linear_model import Ridge, Lasso, ElasticNet, RidgeCV, LassoCV

# === RidgeCV：自动选择最优 alpha ===
ridge_cv = RidgeCV(
    alphas=[0.01, 0.1, 1.0, 10.0, 100.0],
    cv=5,
    scoring='neg_mean_squared_error',
)
ridge_cv.fit(X_train, y_train)
print(f"最优 alpha: {ridge_cv.alpha_}")

# === Lasso：通过 L1 正则化自动特征选择 ===
lasso = Lasso(
    alpha=0.01,
    max_iter=10000,       # 必须够大才能收敛
    selection='cyclic',   # 'cyclic'（循环更新）或 'random'
    random_state=42,
)

# === ElasticNet ===
elastic = ElasticNet(
    alpha=0.01,
    l1_ratio=0.7,  # 0 = 纯 Ridge, 1 = 纯 Lasso
    max_iter=10000,
)

# 查看 Lasso 选择了哪些特征（非零系数）
non_zero = np.where(lasso.coef_ != 0)[0]
print(f"Lasso 选择了 {len(non_zero)} 个特征")
```

### 3.2 树模型家族

#### 3.2.1 Decision Tree → Random Forest → GBDT 的演化逻辑

```
决策树（Decision Tree）
  │ 缺点：高方差，容易过拟合
  │
  ├─→ Bagging（Bootstrap Aggregating）
  │   └─→ 随机森林（Random Forest）
  │       原理：训练多棵决策树（每棵用不同的 bootstrap 样本和特征子集）
  │             最后投票/平均
  │       效果：方差 ↓↓，偏差基本不变
  │       特点：树之间独立，可并行训练
  │
  └─→ Boosting
      └─→ GBDT（Gradient Boosting Decision Tree）
          原理：串行训练决策树，每棵新树拟合前一棵的残差（梯度）
          效果：偏差 ↓↓，方差 ↑（需正则化控制）
          特点：树之间依赖，必须串行训练
```

**Bagging vs Boosting 直观理解：**

- **Bagging（随机森林）** = "三个臭皮匠，顶个诸葛亮"：每个模型可能不完美，但多数投票能消除个别错误。
- **Boosting（GBDT）** = "站在前人的肩膀上进步"：每个新模型专门学习前一个模型犯的错误。

#### 3.2.2 XGBoost 详解

**核心创新点：**

1. **正则化目标函数：** `Obj = Σ L(y, ŷ) + Σ Ω(f)`，其中 `Ω(f) = γT + ½λ Σw²`
   - 传统 GBDT 没有正则化项
   - XGBoost 的正则化显著降低了过拟合风险
2. **二阶泰勒展开近似：** 用 Hessian（二阶导数）指导分裂，比只用梯度更精确
3. **分位数近似（Weighted Quantile Sketch）：** 大数据下高效找分裂点
4. **列采样（Column Subsampling）：** 借鉴随机森林的思路
5. **稀疏感知（Sparsity-aware）：** 自动学习缺失值的最优分裂方向

**关键超参数及调参策略：**

```python
import xgboost as xgb

# ============ 最关键的参数 ============
params = {
    # 核心参数
    'n_estimators': 1000,          # 树的数量（用 early stopping 确定）
    'learning_rate': 0.05,         # ⭐ 最重要的参数！0.01-0.3
    'max_depth': 6,                # ⭐ 树深度，3-10，核心正则化参数
    'min_child_weight': 1,         # 叶片最小权重和（越大越保守）

    # 随机采样参数
    'subsample': 0.8,              # 行采样率
    'colsample_bytree': 0.8,       # 每棵树的列采样率
    'colsample_bylevel': 0.8,      # 每层的列采样率
    'colsample_bynode': 0.8,       # 每个节点的列采样率

    # 正则化参数
    'gamma': 0,                    # 分裂所需的最小损失减少（越大越保守）
    'reg_alpha': 0.1,              # L1 正则化（对抗高维稀疏特征）
    'reg_lambda': 1.0,             # L2 正则化（对抗过拟合的通用利器）

    # 其他
    'tree_method': 'hist',         # 'hist'（直方图加速）或 'gpu_hist'
    'scale_pos_weight': 1.0,       # 类别不平衡
    'random_state': 42,
    'eval_metric': 'auc',
    'early_stopping_rounds': 100,
    'verbosity': 0,
}

model = xgb.XGBClassifier(**params)
```

**XGBoost 五步调参法：**

```
Step 1: 固定学习率 0.1，用 early stopping 确定 n_estimators
Step 2: 调 max_depth 和 min_child_weight（最重要的一步）
Step 3: 调 gamma
Step 4: 调 subsample 和 colsample_bytree
Step 5: 调 reg_alpha 和 reg_lambda（微调）
最后: 降低 learning_rate（如 0.01），增加 n_estimators，重新用 early_stopping
```

#### 3.2.3 LightGBM 详解

**与 XGBoost 的核心区别：**

| 特性 | XGBoost | LightGBM |
|------|---------|----------|
| 树生长策略 | Level-wise（按层生长） | **Leaf-wise（按叶生长）** |
| 分裂点查找 | 预排序 / 分位数近似 | **直方图（Histogram）** |
| 类别特征 | 需手动编码 | 原生支持（标记为 categorical_feature） |
| 缺失值 | 学习最优方向 | 学习最优方向 |
| 并行策略 | 特征并行 | 特征并行 + 数据并行 + 投票并行 |
| 训练速度 | 中 | 快（尤其大数据时快 5-10 倍） |
| 内存占用 | 较大 | 小（直方图压缩 + EFB） |
| 小数据精度 | 略高 | 可能略低（leaf-wise 易过拟合） |

**Leaf-wise vs Level-wise 图解：**

```
Level-wise (XGBoost):          Leaf-wise (LightGBM):
     [根]                         [根]
    /    \                       /    \
  [A]    [B]                   [A]    [B]
  / \    / \                  /      / \
 1   2  3   4                1     [C]  3
                                   / \
                                  2   4
每一层同时扩展               只扩展增益最大的叶子
更平衡，更慢                 不平衡，更快，但需控制深度
```

**LightGBM 核心参数：**

```python
import lightgbm as lgb
```

#### 3.2.4 CatBoost 详解

**CatBoost 的独门绝技：**

1. **Ordered Boosting（有序提升）：** 标准 GBDT 的梯度计算基于整个数据集，会导致**预测偏移（prediction shift）**。CatBoost 用排列组合的方式，每个样本只用"之前"的样本计算残差，消除了这种偏移。
2. **有序目标编码（Ordered Target Encoding）：** 对类别特征进行目标编码时，同样避免了数据泄露。
3. **对称树（Symmetric Trees）：** 所有分裂使用相同的特征（类似决策树桩），预测速度极快。

**CatBoost 核心参数：**

```python
from catboost import CatBoostClassifier, Pool

# 创建数据 Pool（CatBoost 的数据容器）
train_pool = Pool(
    X_train, y_train,
    cat_features=cat_features,  # 类别特征的列名或索引
)

valid_pool = Pool(X_valid, y_valid, cat_features=cat_features)

model = CatBoostClassifier(
    # 核心参数
    iterations=1000,
    learning_rate=0.05,
    depth=6,                      # 树深度（对称树）
    l2_leaf_reg=3.0,              # L2 正则化

    # 随机性控制（防止过拟合）
    random_strength=1.0,          # 分裂打分时加入的随机噪声
    bagging_temperature=0.5,      # 贝叶斯 Bootstrap 的温度参数

    # 类别特征处理
    one_hot_max_size=2,           # 类别数 ≤ 此值用 One-Hot，否则用目标编码

    # 其他
    loss_function='Logloss',      # 损失函数
    eval_metric='AUC',
    early_stopping_rounds=100,
    random_seed=42,
    verbose=0,
    task_type='CPU',              # 'CPU' 或 'GPU'
)

model.fit(train_pool, eval_set=valid_pool)
```

### 3.3 SVM（Support Vector Machine）

SVM 在 Kaggle 中的应用越来越少，但在特定场景下仍有价值。

**核函数选择指南：**

| 核函数 | 适用场景 | 特点 |
|--------|----------|------|
| Linear（线性） | 高维稀疏特征（TF-IDF 文本） | 快、可解释 |
| RBF（高斯） | 低维小数据（<10K 样本） | **最常用**，非线性能力强 |
| Polynomial | 已知特征之间有交互关系 | 参数多，难调 |
| Sigmoid | 类似于浅层神经网络 | 极少使用 |

**SVM 的核心限制：**

- 时间复杂度 O(n²) 到 O(n³)，数据量 > 50K 就很慢
- 对特征缩放极其敏感
- 需要网格搜索 C 和 γ，计算量大
- 不支持概率输出（需 Platt Scaling，不够准确）


### 3.4 KNN（K-Nearest Neighbors）

KNN 是最直观的机器学习算法，但在竞赛中作为模型使用场景有限。

**距离度量选择：**

| 距离 | 适用数据 | 特点 |
|------|----------|------|
| Euclidean（欧氏距离） | 连续数值 | 默认选择 |
| Manhattan（曼哈顿距离） | 高维数据 | 维度诅咒下比欧氏好 |
| Cosine（余弦相似度） | 文本/稀疏向量 | 不关心向量长度，只看方向 |
| Hamming（汉明距离） | 二值特征 | 独热编码的类别特征 |

**维度诅咒（Curse of Dimensionality）：**

当特征维度增加时，KNN 面临根本性困难：
- 高维空间中所有点"看起来都一样远"
- 最近邻不再有意义
- **经验法则：KNN 适合特征维度 < 20**

```python
from sklearn.neighbors import KNeighborsClassifier
from sklearn.preprocessing import StandardScaler

# KNN 也需要标准化！
knn = KNeighborsClassifier(
    n_neighbors=5,                # K 值：小 → 复杂（高方差），大 → 平滑（高偏差）
    weights='distance',           # 'uniform'（等权）或 'distance'（距离倒数加权）
    metric='minkowski',           # 距离度量
    p=2,                          # p=2 是欧氏距离，p=1 是曼哈顿
    n_jobs=-1,
)
```

**KNN 在竞赛中的实际用途：**
- **特征工程：** KNN 可以用来生成"距离最近 N 个邻居的标签均值"作为新特征
- **离群点检测：** 基于距离的异常检测
- **Baseline：** 回归/分类的简单 baseline

### 3.5 神经网络（在表格数据上的应用）

#### 3.5.1 什么时候表格数据该用神经网络

**不用 NN 的情况（90% 的表格竞赛）：**
- 数据量 < 100K
- 特征以连续数值和低基数类别为主
- GBDT 已经能很好地完成任务

**考虑 NN 的情况：**
- 数据量 > 1M（NN 的样本效率开始体现）
- 特征包含高基数类别（>1000 个类别）
- 多任务学习（同时预测多个目标）
- 多模态（表格 + 文本 + 图像需要统一编码）
- GBDT 已经调无可调，需要最后一搏

#### 3.5.2 表格数据的 NN 架构选择

```python
        self.network = nn.Sequential(*layers)
```

1. **BatchNorm > LayerNorm**：表格数据中 BatchNorm 通常更好

### 3.6 集成学习

集成学习是 Kaggle 竞赛的最后冲刺手段。

#### 3.6.1 Bagging vs Boosting vs Stacking 对比

| 方法 | 原理 | 降偏差/降方差 | 并行性 | 提升幅度 |
|------|------|--------------|--------|----------|
| Bagging | 独立训练多个模型，投票/平均 | 降方差 | 可并行 | 小到中 |
| Boosting | 串行训练，每轮关注前一轮的错 | 降偏差 | 不可并行 | 中到大 |
| Stacking | 用多个模型的预测训练 meta-model | 两者皆有 | 第一层可并行 | 中 |
| Blending | Stacking 的简化版（单 hold-out） | 两者皆有 | 第一层可并行 | 小到中 |
| 加权平均 | 对模型预测加权求和 | 降方差 | 可并行 | 小 |

#### 3.6.2 Stacking 的完整实现

Stacking 的核心是通过交叉验证生成"元特征"（meta-features），避免数据泄露。

```python
import numpy as np
from sklearn.model_selection import KFold
from sklearn.linear_model import Ridge
from lightgbm import LGBMRegressor
from xgboost import XGBRegressor
from catboost import CatBoostRegressor

def generate_meta_features(models, X, y, n_splits=5, random_state=42):
    """
    通过 K-Fold 交叉验证生成 meta features（OOF 预测）
    这是 stacking 的核心步骤 —— 避免数据泄露

    Returns:
        oof_preds: (n_samples, n_models) 的矩阵
        models: 在全部训练数据上重新训练的模型列表
    """
    kf = KFold(n_splits=n_splits, shuffle=True, random_state=random_state)
    oof_preds = np.zeros((X.shape[0], len(models)))

    for fold_idx, (train_idx, val_idx) in enumerate(kf.split(X)):
        X_train_fold, X_val_fold = X.iloc[train_idx], X.iloc[val_idx]
        y_train_fold, y_val_fold = y.iloc[train_idx], y.iloc[val_idx]

        for model_idx, (name, model) in enumerate(models):
            model_clone = model.__class__(**model.get_params())
            # 在 fold 训练集上训练
            model_clone.fit(X_train_fold, y_train_fold,
                           eval_set=[(X_val_fold, y_val_fold)],
                           callbacks=[model_clone.__class__.__module__.startswith('lightgbm')
                                      and '__import__("lightgbm").early_stopping(50)'
                                      or None],
                           verbose=False)
            # 在 fold 验证集上预测
            if hasattr(model_clone, 'predict_proba'):
                oof_preds[val_idx, model_idx] = model_clone.predict_proba(X_val_fold)[:, 1]
            else:
                oof_preds[val_idx, model_idx] = model_clone.predict(X_val_fold)

    # 用全部数据重新训练每个模型
    full_models = []
    for name, model in models:
        model_full = model.__class__(**model.get_params())
        model_full.fit(X, y, verbose=False)
        full_models.append(model_full)

    return oof_preds, full_models

# ======== 使用示例 ========
base_models = [
    ('xgb', XGBRegressor(n_estimators=500, learning_rate=0.05, max_depth=5, random_state=42)),
    ('lgb', LGBMRegressor(n_estimators=500, learning_rate=0.05, num_leaves=31, random_state=42)),
```

```python
# 更简洁的 OOF 生成方式
def get_oof_predictions(model_class, params, X, y, X_test, n_splits=5):
    """适用于 LightGBM/XGBoost/CatBoost 的通用 OOF 包装器"""
    from sklearn.model_selection import KFold
    import numpy as np

    kf = KFold(n_splits=n_splits, shuffle=True, random_state=42)
    oof = np.zeros(len(X))
    test_preds = np.zeros(len(X_test))

    for train_idx, val_idx in kf.split(X):
        X_tr, X_val = X.iloc[train_idx], X.iloc[val_idx]
        y_tr, y_val = y.iloc[train_idx], y.iloc[val_idx]

        model = model_class(**params)
        model.fit(X_tr, y_tr,
                  eval_set=[(X_val, y_val)],
                  callbacks=[type(model).__name__ == 'LGBMClassifier'
                             and lightgbm_early_stop(100) or None],
                  verbose=False)

        oof[val_idx] = model.predict_proba(X_val)[:, 1]
        test_preds += model.predict_proba(X_test)[:, 1] / n_splits

    return oof, test_preds
```

#### 3.6.4 加权平均（Weighted Average）

```python
# 最简单但有效的集成方式
from scipy.optimize import minimize

def find_optimal_weights(oof_preds, y_true, metric_func):
    """
    在验证集上通过优化找到最优权重
    metric_func: 返回需要最小化的指标值
    """
    n_models = oof_preds.shape[1]
    
    def loss(weights):
        weights = weights / weights.sum()  # 归一化
        weighted_pred = (oof_preds * weights).sum(axis=1)
        return metric_func(y_true, weighted_pred)
    
    # 初始等权重
    init_weights = np.ones(n_models) / n_models
    bounds = [(0, 1)] * n_models
    constraints = {'type': 'eq', 'fun': lambda w: w.sum() - 1}
    
    result = minimize(loss, init_weights,
                     method='SLSQP',
                     bounds=bounds,
                     constraints=constraints)
    
    return result.x / result.x.sum()
```

### 4.1 完整决策路径（文字版决策树）

```
    │     ├─ 时间序列预测 → Step 2（但 CV 策略不同）

    │     │
    │     └─ 数据噪声大？
    │         ├─ 标签噪声 → 集成方法（Bagging）、增大正则化

    │     │
    │     ├─ 第 1 轮：单模型 + 少量特征工程
    │     │   └─→ 得到一个可靠的 CV 分数作为 benchmark
    │     │
    │     ├─ 第 2 轮：特征工程深入
    │     │   ├─ 构造交互特征、聚合特征、lag 特征

    │     │   └─→ 重新训练，看 CV 提升
    │     │
    │     ├─ 第 3 轮：模型调参
    │     │   ├─ 用 Optuna 或手动搜索关键参数
    │     │   └─→ 再提升 1-3 个点
    │     │
    │     ├─ 第 4 轮：尝试不同模型
    │     │   ├─ 如果用的是 XGBoost → 试 LightGBM 和 CatBoost
    │     │   ├─ 如果数据 > 1M → 试 TabNet 或 MLP
    │     │   └─→ 比较 CV，保留最优的 2-3 个模型
    │     │
    │     └─ 第 5 轮：集成（最后冲刺）
    │         ├─ 不同 GBDT 框架的 ensemble
    │         ├─ 不同超参数配置的 ensemble
    │         ├─ 不同特征子集的 ensemble
    │         └─→ 通常能再提升 0.5-2 个点

    └─→ 最终方案确定
```

## 5. 超参数调优策略

### 5.1 三种调参方法对比

| 方法 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **Grid Search** | 穷举所有参数组合 | 可复现、简单 | 维度灾难，参数多时组合爆炸 | 1-2 个参数微调 |
| **Random Search** | 随机采样参数组合 | 比 Grid 效率高、可覆盖更多值 | 不够系统 | 3-5 个参数 |
| **Bayesian Optimization** | 根据历史结果智能选择下一组参数 | **效率最高**、自动收敛 | 实现复杂 | ⭐ 首选 |

**为什么不用 Grid Search：**
- 调 5 个参数，每个 10 个值 → 10^5 = 100,000 种组合，根本跑不完
- 大部分参数组合是无效的（在已知劣质区域浪费计算）
- Random Search 可以在同样的计算量下探索更多的参数"水平"

**早停的核心原理：**

```
验证分数
  ↑
  │                     最佳点
  │                    ╱    ╲___
  │                  ╱          ╲_____
  │               ╱                   ╲_____  过拟合开始
  │             ╱
  │   训练分数 ╱
  │ ╱╲╱╲╱╲╱╲╱╲╱
  │╱

              ↑            ↑
```

### 6.1 交叉验证策略选择

错误的 CV 策略会给出错误的"线下分数"，导致你选了一个线上崩掉的模型。

| CV 策略 | 什么时候用 | 什么时候不用 |
|---------|-----------|-------------|
| **KFold** | 通用的默认选择，回归任务 | 分类任务更推荐 StratifiedKFold |
| **StratifiedKFold** | ⭐ **分类任务的默认选择** | 回归任务不能用 |
| **GroupKFold** | 有 group 信息且组之间独立 | 无 group 或 group 泄露信息 |
| **TimeSeriesSplit** | ⭐ **时间序列的默认选择** | 非时间序列数据 |
| **StratifiedGroupKFold** | 分类 + 有 group | sklearn 没有内置，需自己写 |
| **LeaveOneOut** | 数据极小（<100） | 计算量太大 |
| **RepeatedKFold** | 需要更稳定的估计 | 计算资源有限 |

```python
    return fold_assignments
```

### 6.2 评估指标选择指南

选择合适的评估指标和选择合适的模型同样重要。

#### 分类指标

| 指标 | 公式/含义 | 使用场景 | 对不平衡敏感？ |
|------|-----------|----------|:---:|
| Accuracy | (TP+TN) / Total | 均衡分类 | ✅ 是 |
| **AUC-ROC** | ROC 曲线下面积 | ⭐ 最大区分度，最通用 | ❌ 否（推荐） |
| **LogLoss** | -Σ y·log(p) | 需要校准的概率（保险定价） | 中等 |
| **F1 Score** | 2PR/(P+R) | 正类非常重要（欺诈检测） | ❌ 否 |
| **Precision** | TP / (TP+FP) | 减少误报（垃圾邮件过滤） | ❌ 否 |
| **Recall** | TP / (TP+FN) | 减少漏报（疾病筛查） | ❌ 否 |
| MCC | - | 对极端不平衡公平 | ❌ 否 |
| QWK | Quadratic Weighted Kappa | 有序分类（评分预测） | ❌ 否 |

#### 回归指标

| 指标 | 公式 | 性质 | 使用场景 |
|------|------|------|----------|
| **RMSE** | √(Σ(ŷ-y)²/n) | 对大误差惩罚大 | 一般回归（默认） |
| **MAE** | Σ\|ŷ-y\|/n | 对大误差惩罚小于 RMSE | 离群值多的场景 |
| **RMSLE** | √(Σ(log(ŷ+1)-log(y+1))²/n) | 惩罚相对误差 | 目标跨多个数量级 |
| **MAPE** | Σ\|(ŷ-y)/y\|/n × 100% | 相对误差百分比 | 关注相对误差（需 y>0） |
| **R²** | 1 - SS_res / SS_tot | 解释方差比例 | 评估模型解释能力 |
| SMAPE | 对称 MAPE | 处理零值 | 预测值有零时 |

```python
from sklearn.metrics import (
    accuracy_score, roc_auc_score, log_loss,
    f1_score, precision_score, recall_score,
    mean_squared_error, mean_absolute_error,
    r2_score, cohen_kappa_score,
)
import numpy as np

# ===== 常用指标的自定义函数 =====

def rmsle(y_true, y_pred):
    """Root Mean Squared Logarithmic Error"""
    return np.sqrt(np.mean(np.square(np.log1p(y_pred) - np.log1p(y_true))))

def mape(y_true, y_pred):
    """Mean Absolute Percentage Error"""
    return np.mean(np.abs((y_true - y_pred) / y_true)) * 100

# ===== Kaggle 风格的安全评分函数 =====
def safe_auc(y_true, y_pred):
    """安全的 AUC 计算（处理只有一类的情况）"""
    if len(np.unique(y_true)) < 2:
        return np.nan
    return roc_auc_score(y_true, y_pred)

# ===== 竞赛常见的评分函数包装 =====
def competition_score(y_true, y_pred, metric='auc'):
    """统一的评估函数接口"""
    metrics = {
        'auc': roc_auc_score,
        'logloss': log_loss,
        'rmse': lambda y, p: np.sqrt(mean_squared_error(y, p)),
        'mae': mean_absolute_error,
        'rmsle': rmsle,
        'f1': f1_score,
        'qwk': cohen_kappa_score,  # 需要四舍五入到整数
    }
    func = metrics.get(metric)
    if func is None:
        raise ValueError(f"Unknown metric: {metric}")
    return func(y_true, y_pred)
```

### 6.3 线下验证与线上得分的一致性

这是 Kaggle 竞赛中最让人头疼的问题之一：**线下 CV 高了，线上分数却掉了**（经常被称为 "shake-up"）。

**线下/线上不一致的主要原因及对策：**

| 原因 | 表现 | 对策 |
|------|------|------|
| **数据泄露（Data Leakage）** | 线下极高，线上崩 | 检查 CV 中是否有时间穿越、group 泄露 |
| **过拟合验证集** | 线下稳定提升，线上不涨 | CV 标准化操作、用 hold-out 做最终检验 |
| **分布偏移（Test ≠ Train）** | 线下/线上有稳定差距 | Adversarial Validation、特征分布可视化 |
| **CV 策略不匹配任务** | 线下波动大 | 确认 CV 策略正确（时间/group/分层） |
| **评估指标估计偏差** | 评估方法有 bug | 用 sklearn 的 metrics 交叉验证 |

**Adversarial Validation（对抗验证）检测分布偏移：**

```python
# 检测训练集和测试集是否有分布偏移
def adversarial_validation(train_df, test_df):
    """
    如果 AUC ≈ 0.5 → 训练集和测试集分布一致（理想）
    如果 AUC > 0.6 → 存在明显的分布偏移，需要小心！
    """
    # 创建标签：train=1, test=0
    combined = pd.concat([train_df, test_df], ignore_index=True)
    y_adv = np.array([1]*len(train_df) + [0]*len(test_df))
    
    # 训练一个分类器区分 train/test
    from sklearn.model_selection import cross_val_score
    clf = lgb.LGBMClassifier(n_estimators=100, learning_rate=0.1, verbose=-1)
    scores = cross_val_score(clf, combined, y_adv, cv=5, scoring='roc_auc')
    
    auc = np.mean(scores)
    if auc > 0.65:
        print(f"⚠️  发现严重分布偏移！AUC={auc:.3f}")
        print("建议：1) 用对抗验证找出偏移最大的特征  2) 考虑用更保守的模型")
    elif auc > 0.55:
        print(f"⚠️  存在轻微分布偏移，AUC={auc:.3f}")
    else:
        print(f"✅ 分布基本一致，AUC={auc:.3f}，线下分数可信")
    
    return auc
```

**稳定线下分数的最佳实践：**

```python
# 完整的验证 Pipeline
def robust_validation(model, X, y, n_splits=5, n_repeats=3, eval_metric='auc'):
    """
    多重 CV 确保线下分数稳定可靠
    """
    cv = RepeatedStratifiedKFold(
        n_splits=n_splits,
        n_repeats=n_repeats,
        random_state=42,
    )
    
    scores = []
    for fold, (train_idx, val_idx) in enumerate(cv.split(X, y)):
        X_tr, X_val = X.iloc[train_idx], X.iloc[val_idx]
        y_tr, y_val = y.iloc[train_idx], y.iloc[val_idx]
        
        # 每个 fold 独立训练（不要用之前的模型状态）
        model_fold = model.__class__(**model.get_params())
        model_fold.fit(X_tr, y_tr, verbose=False)
        
        if hasattr(model_fold, 'predict_proba'):
            y_pred = model_fold.predict_proba(X_val)[:, 1]
        else:
            y_pred = model_fold.predict(X_val)
        
        score = competition_score(y_val, y_pred, eval_metric)
        scores.append(score)
    
    # 报告均值和标准差
    print(f"CV {eval_metric}: {np.mean(scores):.4f} (±{np.std(scores):.4f})")
    print(f"  各 fold 分数: {[f'{s:.4f}' for s in scores]}")
    
    # 如果 fold 间标准差过大（>0.01），说明模型对数据划分敏感
    if np.std(scores) > 0.01:
        print("  ⚠️ 标准差较大，考虑：1) 增加 n_splits  2) 更稳定的模型")
    
    return np.mean(scores), np.std(scores)
```

## 7. 实战模型选择 Checklist

当开始一个新 Kaggle 竞赛时，按以下 checklist 逐步推进：

### Stage 1：数据理解（前 1-2 小时）

- [ ] 读取数据，理解每列的含义和目标变量
- [ ] 检查数据规模：样本数和特征数
- [ ] 检查数据类型：数值、类别、文本、图像？
- [ ] 检查缺失值比例（每列）
- [ ] 检查目标变量的分布（偏斜程度、类别比例）
- [ ] 检查 train/test 的行数比
- [ ] 跑 Adversarial Validation（检测分布偏移）

### Stage 2：快速 Baseline（30 分钟内）

- [ ] **分类任务：** Logistic Regression / 浅层 RandomForest
- [ ] **回归任务：** Ridge / 浅层 RandomForest
- [ ] **时间序列：** Naive forecast / 简单 Lag + LR
- [ ] 确保 CV 策略正确（时间序列用 TimeSeriesSplit，分类用 StratifiedKFold）
- [ ] 确认评估指标实现正确（和竞赛打分函数一致）
- [ ] 得到一个 **benchmark CV 分数**（后续所有改进以此为基准）

### Stage 3：特征工程 + 模型初选（几小时到 1 天）

- [ ] 基础特征工程（缺失值处理、编码、缩放）
- [ ] 创建有意义的衍生特征（交互、聚合、比率）
- [ ] 根据数据规模和数据质量选择候选模型（参照本文第 2 节）
- [ ] 训练 2-3 个不同模型（如 XGBoost + LightGBM + CatBoost）
- [ ] 比较 CV 分数和训练速度
- [ ] 检查特征重要性（删除无用特征、添加缺失指示特征）

### Stage 4：深度优化（1-3 天）

- [ ] 深入特征工程（domain knowledge、外部数据）
- [ ] 超参数调优（Optuna Bayesian Search）
- [ ] 尝试不同 CV 策略确认线下分数稳定性
- [ ] 特征选择（基于重要性分数 + CV）
- [ ] 比较单模型在不同特征子集上的表现
- [ ] 多模型比较的正式记录（记录每次实验的参数和 CV 分数）

### Stage 5：集成冲刺（最后 1-2 天）

- [ ] 选择 3-5 个多样性足够的最优模型（不同算法、参数、或特征集）
- [ ] OOF 预测生成 meta features
- [ ] 训练简单 meta model（Ridge/Logistic Regression）
- [ ] 比较 Stacking vs 简单加权平均
- [ ] 评估集成的 CV 提升（通常 0.5-2 个点）
- [ ] 确认无数据泄露（OOF 生成过程正确）

### Stage 6：最终检查

- [ ] 重新检查数据泄露（有没有使用未来信息？有没有 group 泄露？）
- [ ] 检查预测值的分布是否合理（没有异常值、没有全 0 或全 1）
- [ ] 测试集特征分布是否与训练集基本一致
- [ ] 生成 submission 文件，检查格式
- [ ] 保存模型和关键中间文件（model checkpoint、OOF 预测、特征重要性）
