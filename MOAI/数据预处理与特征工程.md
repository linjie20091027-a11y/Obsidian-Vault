## 1. 数据探索（EDA）的核心框架

### 1.1 拿到数据的第一步做什么

拿到 `train.csv` 和 `test.csv` 后，按以下清单执行：

1. **合并 train 和 test**（统一预处理，避免不一致）
2. **查看 shape** —— 样本量、特征数决定后续策略
3. **查看 dtypes** —— 发现类型错误（如数值被读成 object）
4. **describe()** —— 快速发现异常值、常量特征
5. **nunique() / 缺失率** —— 识别高基数特征、决定是否可直接删除
6. **偏度(skewness)** —— 决定变换策略
7. **目标变量分布** —— 决定评估指标和建模策略

```python
import pandas as pd
import numpy as np

# ===== 第一步：加载并合并 =====
train = pd.read_csv('train.csv')
test  = pd.read_csv('test.csv')

target = train['target']  # 保存目标变量
train.drop('target', axis=1, inplace=True)

# 添加标记列，方便最后拆分
train['_is_train'] = True
test['_is_train']  = False

df = pd.concat([train, test], axis=0, ignore_index=True)
print(f"合并后 shape: {df.shape}")
```

### 1.2 必看指标

```python
# ===== 基本信息 =====
print(f"Shape: {df.shape}")
print(f"Memory: {df.memory_usage(deep=True).sum() / 1024**2:.2f} MB")

# ===== 类型分布 =====

desc['missing%']  = (df.isnull().mean() * 100).round(2)

# ===== 快速筛选问题特征 =====
# 缺失率 > 60% 的特征

print(f"缺失率 > 60%: {high_miss}")

# 唯一值只有 1 的特征（常量列）

print(f"常量列: {constant_cols}")

# |skew| > 1 的高度偏态特征

# 高基数类别特征（>100 个唯一值，但又是 object）

    c for c in df.select_dtypes('object').columns
    if df[c].nunique() > 100
]
print(f"高基数类别: {high_cardinality}")
```

### 1.4 相关性分析 —— Pearson vs Spearman vs Kendall

| 方法 | 适用场景 | 要求 |
|------|----------|------|
| **Pearson** | 线性关系、近似正态 | 两个变量均为连续且近似正态 |
| **Spearman** | 单调关系（不要求线性） | 排序数据，对异常值鲁棒 |
| **Kendall** | 小样本、有大量并列值 | 序数数据，比 Spearman 更稳健但更慢 |

**实战建议**：默认先用 Spearman，因为真实数据很少严格满足正态假设。Pearson 容易被异常值拉偏。

```python
# ===== 三种相关性系数对比 =====
corr_pearson  = df.select_dtypes(np.number).corr(method='pearson')
corr_spearman = df.select_dtypes(np.number).corr(method='spearman')
corr_kendall  = df.select_dtypes(np.number).corr(method='kendall')  # 慢，小数据集用

# ===== 找出与目标变量 top-k 相关的特征 =====
target_corr = corr_spearman['target'].abs().sort_values(ascending=False)
print(f"Top-10 与 target 最相关的特征:\n{target_corr.head(11).iloc[1:]}")  # 去掉 target 自身

# ===== 找高度相关的特征对（共线性检测） =====
# 上三角矩阵，|corr| > 0.95
import itertools
high_corr_pairs = []
for col1, col2 in itertools.combinations(corr_spearman.columns, 2):
    if abs(corr_spearman.loc[col1, col2]) > 0.95:
        high_corr_pairs.append((col1, col2, corr_spearman.loc[col1, col2]))
print(f"高度相关特征对 (|r|>0.95): 共 {len(high_corr_pairs)} 对")
```

## 2. 偏态数据（Skewed Data）的处理

### 2.1 什么是偏态

**定义**：数据分布不对称，一边的尾部比另一边更长。

| 偏态类型 | 别称 | skewness 符号 | 特征 | 典型场景 |
|----------|------|---------------|------|----------|
| **右偏（正偏）** | Positive Skew | skew > 0 | 长尾在右侧，均值 > 中位数 | 收入、房价、点击量 |
| **左偏（负偏）** | Negative Skew | skew < 0 | 长尾在左侧，均值 < 中位数 | 考试成绩、寿命 |

**判断标准**：

| skewness 绝对值 | 程度 | 处理建议 |
|:---:|:---:|------|
| < 0.5 | 近似对称 | 通常不需处理 |
| 0.5 ~ 1.0 | 中等偏态 | 考虑变换 |
| > 1.0 | 高度偏态 | 强烈建议变换 |

### 2.2 为什么偏态数据会影响模型

| 影响维度 | 机制 |
|----------|------|
| **线性模型** | 假设残差服从正态分布，偏态导致系数估计有偏、置信区间不准 |
| **梯度下降** | 偏态特征的尺度差异大，导致梯度更新不平衡，收敛慢 |
| **距离度量** | KNN/KMeans 中，偏态特征的主导尾部使距离计算失真 |
| **神经网络** | 偏态输入导致激活函数饱和（如 sigmoid 的饱和区），梯度消失 |

### 2.3 偏态数据的变换方法

#### (1) Log 变换 —— 最常用，右偏首选

适用条件：数据 ≥ 0（不含 0 用 log1p，含 0 通常先加常数或用 log1p）。

```python
# ===== Log 变换 =====
from numpy import log1p, expm1

# log1p(x) = log(1 + x)，天然处理 x=0 的情况
df['feat_log'] = np.log1p(df['original_feat'])

# 逆变换
df['feat_original'] = np.expm1(df['feat_log'])

# ===== 可视化变换前后对比 =====
fig, axes = plt.subplots(2, 2, figsize=(12, 10))
sns.histplot(df['original_feat'], kde=True, ax=axes[0, 0])
axes[0, 0].set_title(f'变换前 (skew={df["original_feat"].skew():.2f})')
sns.histplot(df['feat_log'], kde=True, ax=axes[0, 1])
axes[0, 1].set_title(f'Log变换后 (skew={df["feat_log"].skew():.2f})')
stats.probplot(df['original_feat'], dist="norm", plot=axes[1, 0])
stats.probplot(df['feat_log'], dist="norm", plot=axes[1, 1])
plt.show()
```

#### (2) Box-Cox 变换 —— 自动寻最优 λ

要求：数据 **严格 > 0**。

```python
from scipy.stats import boxcox

# Box-Cox：自动寻找使数据最接近正态的 λ
df['feat_boxcox'], lambda_opt = boxcox(df['original_feat'].fillna(0.001))
print(f"最优 λ = {lambda_opt:.4f}")
print(f"变换后 skew = {df['feat_boxcox'].skew():.4f}")

# λ 的含义：
# λ = 0  → 等效 log 变换
# λ = 1  → 不变换
# λ = 0.5 → 等效平方根变换
# λ = -1 → 等效倒数变换
```

#### (3) Yeo-Johnson 变换 —— 可处理负值

Box-Cox 的推广，**不要求数据为正**。

```python
from sklearn.preprocessing import PowerTransformer

pt = PowerTransformer(method='yeo-johnson', standardize=False)
df['feat_yj'] = pt.fit_transform(df[['original_feat']])
print(f"Yeo-Johnson 变换后 skew = {df['feat_yj'].skew():.4f}")

# 注意：PowerTransformer(standardize=True) 会在变换后自动做 zero-mean unit-variance
# 如果你只需要消除偏态不要标准化，设 standardize=False
```

#### (4) 平方根变换 —— 轻度右偏

```python
df['feat_sqrt'] = np.sqrt(df['original_feat'])
# 适用：偏度 0.5~1.0，不想改变数据性质太多
```

#### (5) 倒数变换 —— 极端右偏

```python
df['feat_reciprocal'] = 1.0 / (df['original_feat'] + 1e-8)
# 注意：会颠倒大小关系，需谨慎解释
```

#### (6) 分位数变换 —— 强制正态/均匀

```python
from sklearn.preprocessing import QuantileTransformer

# output_distribution='normal'：强制映射为正态分布
qt_norm = QuantileTransformer(n_quantiles=1000, output_distribution='normal', random_state=42)
df['feat_quantile_norm'] = qt_norm.fit_transform(df[['original_feat']])

# output_distribution='uniform'：强制映射为 [0,1] 均匀分布
qt_unif = QuantileTransformer(n_quantiles=1000, output_distribution='uniform', random_state=42)
df['feat_quantile_unif'] = qt_unif.fit_transform(df[['original_feat']])

# ⚠ 注意：
# 1. 分位数变换基于排序，受离群值影响较小
# 2. n_quantiles < 样本数时，会起到平滑/降噪作用
# 3. 测试集必须用训练集 fit 的 transformer 来 transform
# 4. 变换后有信息泄漏风险（因为用了排序信息），在时序数据中慎用
```

#### 对比表：各种变换一览

| 变换 | 适用条件 | 优点 | 缺点 |
|------|----------|------|------|
| **Log (log1p)** | 数据 ≥ 0，右偏 | 简单直观，可逆，零值天然处理 | 仅处理非负值 |
| **Box-Cox** | 数据 > 0，右偏 | 自动找最优 λ，精确 | 要求严格正值 |
| **Yeo-Johnson** | 任意实数 | 可处理负值和零，最通用 | 计算稍慢 |
| **Square Root** | 数据 ≥ 0，轻度右偏 | 简单，比 log 温和 | 只能处理轻度偏态 |
| **Reciprocal** | 数据 > 0，极端右偏 | 强力压缩长尾 | 颠倒顺序，不太可解释 |
| **QuantileTransformer** | 任意 | 不受离群值影响，效果极致 | 有排序信息泄漏风险，非可逆 |

### 2.4 目标变量偏态的特殊处理

回归竞赛中，target 右偏是最常见场景之一（房价、销售额、等待时间）。

```python
# ===== 完整的 Target 变换 Pipeline =====
from sklearn.model_selection import KFold
from sklearn.metrics import mean_squared_error

# Step 1: 对训练集的 target 做 log 变换
y_train_log = np.log1p(y_train)

# Step 2: 用变换后的 target 训练模型
model = RandomForestRegressor()
model.fit(X_train, y_train_log)

# Step 3: 预测（得到 log 尺度）
y_pred_log = model.predict(X_test)

# Step 4: 逆变换还原
y_pred = np.expm1(y_pred_log)

# Step 5: 评估
rmse = mean_squared_error(y_test, y_pred, squared=False)
print(f"RMSE = {rmse:.4f}")

# ===== 使用 TransformedTargetRegressor 自动化 =====
from sklearn.compose import TransformedTargetRegressor
from sklearn.preprocessing import FunctionTransformer

ttr = TransformedTargetRegressor(
    regressor=RandomForestRegressor(),
    func=np.log1p,           # 变换函数
    inverse_func=np.expm1     # 逆变换函数
)
ttr.fit(X_train, y_train)
y_pred = ttr.predict(X_test)
```

**重要：评估指标的变化**

| 场景 | 推荐指标 | 原因 |
|------|----------|------|
| 原始 target 空间 | RMSE, MAE | 直观，可解释 |
| log 变换后 target 空间 | RMSLE | 惩罚比例误差而非绝对误差 |
| 竞赛常见 | 先确认赛题用的是 RMSE 还是 RMSLE | 若用 RMSLE，直接对 target 做 log1p 训练 |

```python
# RMSLE 的 numpy 实现
def rmsle(y_true, y_pred):
    return np.sqrt(np.mean((np.log1p(y_true) - np.log1p(y_pred)) ** 2))
```

## 3. 极端值（Outliers）的处理

### 3.1 什么是极端值

极端值 = 明显偏离数据整体分布的观测。可能的来源：测量误差、数据录入错误、自然极端事件、数据集的合法变异。

**核心理念：不要见极端值就删！** 先在业务/物理层面判断，再决定策略。

### 3.2 极端值的检测方法

#### (1) IQR 法 —— 最常用

```python
def detect_outliers_iqr(series, factor=1.5):
    """IQR 法检测离群值"""
    Q1 = series.quantile(0.25)
    Q3 = series.quantile(0.75)
    IQR = Q3 - Q1
    lower = Q1 - factor * IQR
    upper = Q3 + factor * IQR
    outliers = series[(series < lower) | (series > upper)]
    return lower, upper, outliers

for col in ['feat_a', 'feat_b', 'feat_c']:
    low, up, outs = detect_outliers_iqr(df[col])
    print(f"{col}: 范围 [{low:.2f}, {up:.2f}], 离群值 {len(outs)} 个 ({len(outs)/len(df)*100:.2f}%)")
```

#### (2) Z-score 法

```python
from scipy.stats import zscore

def detect_outliers_zscore(series, threshold=3):
    """Z-score 法检测离群值"""
    z = np.abs(zscore(series, nan_policy='omit'))
    return z > threshold

outlier_mask = detect_outliers_zscore(df['feat_a'])
print(f"Z-score 法检出 {outlier_mask.sum()} 个离群值")
```

#### (3) Modified Z-score（MAD）—— 更鲁棒

标准 Z-score 使用均值和标准差，这两者本身就被离群值拉偏。MAD 使用中位数和 MAD，更鲁棒。

```python
def detect_outliers_mad(series, threshold=3.5):
    """Modified Z-score (MAD) 检测离群值"""
    median = series.median()
    mad = np.median(np.abs(series - median))
    # 乘以 1.4826 使 MAD 与标准差在正态分布下一致
    modified_z = 0.6745 * (series - median) / mad
    return np.abs(modified_z) > threshold

outlier_mask_mad = detect_outliers_mad(df['feat_a'])
print(f"MAD 法检出 {outlier_mask_mad.sum()} 个离群值")
```

#### (4) Isolation Forest —— 基于树的异常检测

```python
from sklearn.ensemble import IsolationForest

# contamination: 估算的异常值比例
iso = IsolationForest(contamination=0.05, random_state=42, n_estimators=100)
numeric_data = df.select_dtypes(np.number).dropna()
outlier_pred = iso.fit_predict(numeric_data)  # -1 = 异常, 1 = 正常
print(f"IsolationForest 检出 {(outlier_pred == -1).sum()} 个离群值")
```

#### (5) DBSCAN —— 基于密度的异常检测

```python
from sklearn.cluster import DBSCAN
from sklearn.preprocessing import StandardScaler

# DBSCAN: 标记为 -1 的即为噪声点（离群值）
X_scaled = StandardScaler().fit_transform(df.select_dtypes(np.number).dropna())
dbscan = DBSCAN(eps=1.5, min_samples=10)
labels = dbscan.fit_predict(X_scaled)
print(f"DBSCAN 检出 {(labels == -1).sum()} 个噪声点")
```

#### (6) LOF（Local Outlier Factor）

```python
from sklearn.neighbors import LocalOutlierFactor

# LOF: 计算每个点的"局部离群因子"，值 > 1 表示可能是离群点
lof = LocalOutlierFactor(n_neighbors=20, contamination=0.05)
lof_labels = lof.fit_predict(X_scaled)
lof_scores = -lof.negative_outlier_factor_  # 越大越异常
print(f"LOF 检出 {(lof_labels == -1).sum()} 个离群值")
```

### 3.3 极端值的处理策略 —— 决策流程图

```
发现极端值
    │
    ├── 是否数据录入错误 / 物理不可能？ → 是 → 删除
    │
    ├── 是否使用树模型（RF/XGB/LGB/CatBoost）？
    │       └── 是 → 通常不需要处理，树模型对离群值不敏感
    │
    ├── 是否使用线性模型 / 神经网络？
    │       ├── 极端值占比 < 1% → 截断（Winsorize）
    │       ├── 极端值占比 1%~5% → 鲁棒缩放（RobustScaler）
    │       └── 极端值占比 > 5% → 分箱 或 单独建模
    │
    └── 是否使用距离模型（KNN/SVM）？
            ├── 必须处理！ → 鲁棒缩放 + 截断
            └── 或直接用树模型替代
```

#### (A) 截断（Winsorize / Clip）

```python
from scipy.stats.mstats import winsorize

# 将 Series 中两端 1% 的值截断到 1% 和 99% 分位
df['feat_winsor'] = winsorize(df['feat_a'], limits=(0.01, 0.01))

# 等价的手动实现
lower_bound = df['feat_a'].quantile(0.01)
upper_bound = df['feat_a'].quantile(0.99)
df['feat_clip'] = df['feat_a'].clip(lower_bound, upper_bound)

print(f"截断前: max={df['feat_a'].max():.2f}, min={df['feat_a'].min():.2f}")
print(f"截断后: max={df['feat_clip'].max():.2f}, min={df['feat_clip'].min():.2f}")
```

#### (B) 分箱（Binning）

```python
# 等频分箱：每个箱包含大致相同数量的样本
```

### 3.4 极端值处理注意事项

#### 不同模型对极端值的敏感度

| 模型 | 敏感度 | 原因 | 建议 |
|------|:---:|------|------|
| **线性回归** | 极高 | 最小二乘目标函数是平方误差，极端值残差被平方放大 | 必须处理 |
| **逻辑回归** | 高 | 极端特征值主导 sigmoid 输入 | 建议处理 |
| **KNN** | 极高 | 基于距离，极端值直接主导近邻 | 必须处理 |
| **SVM** | 高 | 极端值可能成为支持向量 | 建议处理 |
| **神经网络** | 中-高 | 极端输入导致神经元饱和 | 需归一化 |
| **决策树** | 低 | 只关心分裂点的排序 | 可不处理 |
| **随机森林** | 低 | 多棵树平均，且基于树 | 可不处理 |
| **XGBoost/LightGBM/CatBoost** | 低 | 梯度提升树天然鲁棒 | 可不处理 |
| **Naive Bayes** | 中 | 极端值扭曲概率估计 | 建议处理 |

#### ⚠ 关键原则：fit 只在训练集上做

```python
from sklearn.preprocessing import RobustScaler

# ✅ 正确做法
train_idx = df['_is_train'] == True

X_test  = df.loc[~train_idx, numeric_cols]

# ❌ 错误做法 —— 会导致数据泄漏

X_all_scaled = scaler.fit_transform(df[numeric_cols])  # 用测试集信息 fit
```

```python
# ===== 统计填充 =====

for col in df.select_dtypes(np.number).columns:
    df[col].fillna(df[col].median(), inplace=True)

# 类别列用众数
for col in df.select_dtypes('object').columns:
    df[col].fillna(df[col].mode()[0] if not df[col].mode().empty else 'MISSING', inplace=True)

# 方法2: IterativeImputer (MICE)

from sklearn.impute import IterativeImputer

# 记得先 StandardScaler 再填充，填充后再用 RobustScaler 等
```

## 5. 特征缩放与编码

### 5.1 数值特征缩放

| Scaler | 公式 | 适用场景 | 对离群值 |
|--------|------|----------|:---:|
| **StandardScaler** | (x - μ) / σ | PCA、线性模型、SVM、神经网络 | 敏感 |
| **MinMaxScaler** | (x - min) / (max - min) | 神经网络（需要 0~1 输入）、图像 | 敏感 |
| **RobustScaler** | (x - median) / IQR | 有离群值的线性模型 | 鲁棒 |
| **MaxAbsScaler** | x / max(\|x\|) | 稀疏数据 | 中等 |

**哪些模型需要缩放？**

| 模型 | 需要缩放？ | 推荐 Scaler |
|------|:---:|------|
| 线性/逻辑回归 | ✅ 需要 | StandardScaler 或 RobustScaler |
| SVM（RBF核） | ✅ 必须 | StandardScaler |
| KNN | ✅ 必须 | StandardScaler |
| PCA | ✅ 必须 | StandardScaler |
| 神经网络 | ✅ 必须 | MinMaxScaler 或 StandardScaler |
| 决策树/RF/XGB/LGBM/CatBoost | ❌ 不需要 | — |
| Naive Bayes | 看变体 | Gaussian NB 不需要 |

### 5.2 类别特征编码 —— 决策表

| 编码方式 | 适用场景 | 输出维度 | 缺点 |
|----------|----------|:---:|------|
| **One-Hot** | 低基数(<10)无序类别 | n_categories | 高基数导致维数爆炸 |
| **Label/Ordinal** | 有序类别，或树模型 | 1 | 引入虚假的顺序关系 |
| **Target Encoding** | 高基数类别，有 target | 1 | 极易过拟合，需要 KFold |
| **Frequency Encoding** | 高基数类别，快速基线 | 1 | 可能丢失区分信息 |
| **Binary Encoding** | 中基数(10~100) | log2(n) | 比 One-Hot 紧凑，树模型友好 |
| **CatBoost Encoding** | 高基数，CatBoost 首选 | 1 | 只能用于 CatBoost |

```python
# ===== Target Encoding (带 KFold 防过拟合) =====
from sklearn.model_selection import KFold
import numpy as np

def target_encode_kfold(train, test, cat_col, target_col, n_splits=5, smoothing=10):
    """
    使用 KFold 进行 Target Encoding，有效防止过拟合。
    smoothing: 平滑参数（该类别样本数少时向全局均值回归）
    """
    global_mean = train[target_col].mean()

    # 用编码结果填充
    train[f'{cat_col}_te'] = 0
    test[f'{cat_col}_te'] = 0

    # KFold 编码训练集
    kf = KFold(n_splits=n_splits, shuffle=True, random_state=42)
    for train_idx, val_idx in kf.split(train):
        fold_train = train.iloc[train_idx]
        fold_val   = train.iloc[val_idx]

        # 计算每个类别的均值（带平滑）
        means = fold_train.groupby(cat_col)[target_col].agg(['mean', 'count'])
        means['smoothed'] = (means['count'] * means['mean'] + smoothing * global_mean) / (means['count'] + smoothing)

        # 填充验证集
        train.loc[train.index[val_idx], f'{cat_col}_te'] = \
            fold_val[cat_col].map(means['smoothed'])

    # 编码测试集：用整个训练集的均值
    means_all = train.groupby(cat_col)[target_col].agg(['mean', 'count'])
    means_all['smoothed'] = (means_all['count'] * means_all['mean'] + smoothing * global_mean) / (means_all['count'] + smoothing)
    test[f'{cat_col}_te'] = test[cat_col].map(means_all['smoothed']).fillna(global_mean)

    return train, test

# 使用
train_encoded, test_encoded = target_encode_kfold(train, test, 'category_col', 'target', n_splits=5)

# ===== Frequency Encoding =====
feat_freq = df['cat_col'].value_counts() / len(df)
df['cat_col_freq'] = df['cat_col'].map(feat_freq)

# ===== Binary Encoding =====

df_binary = bin_encoder.fit_transform(df['cat_col'])
```

```python
# 移除与其他特征高度相关的冗余特征
def remove_high_corr(df, threshold=0.95):
    """移除相关系数 > threshold 的特征对中，与 target 相关性较低的那个"""
    corr_matrix = df.corr(method='spearman').abs()
```

## 7. 数据泄漏（Data Leakage）防范

### 7.1 什么是数据泄漏

数据泄漏 = 训练过程中使用了测试集（或未来）的信息，导致验证分数虚高，但实际泛化能力差。

**典型后果**：CV 0.95 → LB 0.65，即所谓的"Shake Up"。

### 7.2 常见泄漏场景及解决方法

| 泄漏场景 | 示例 | 解决方案 |
|----------|------|----------|
| **在 split 前做预处理** | 在 train_test_split 前做 StandardScaler.fit | 先 split，再 fit_transform train，再 transform test |
| **Target Encoding 不用 KFold** | 直接用全量 target mean 做编码 | 用 KFold Target Encoding |
| **缺失值填充用全局统计** | 用全量数据的均值填充 | KFold 内部分别计算填充值 |
| **过采样后再 split** | 先 SMOTE 再 train_test_split | 先 split，再只在训练集上 SMOTE |
| **特征选择用全量数据** | 用全量数据的相关系数筛特征 | 在 CV 的每一折内做特征选择 |
| **时序数据中的未来信息** | 用未来日期的价格预测当前 | 严格遵守时间顺序，用 TimeSeriesSplit |
| **ID 泄漏** | 行 ID 直接编码了 target 信息 | 检查 ID 是否有序、是否有规律 |
| **Group 泄漏** | 同一用户的多条记录既在 train 又在 test | 用 GroupKFold 确保同组不出现在不同 fold |
| **数据增强不当** | 增强的图片和原图分到了 train 和 val | 确保同一原图的增强版本在同一 fold |

### 7.3 完整的防泄漏 Pipeline

```python
from sklearn.model_selection import KFold, train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.impute import SimpleImputer

# ===== 正确的验证流程 =====
def leak_free_pipeline(X, y, n_splits=5):
    """
    每一折内部独立做预处理，彻底杜绝数据泄漏
    """
    kf = KFold(n_splits=n_splits, shuffle=True, random_state=42)
    scores = []

    for fold, (train_idx, val_idx) in enumerate(kf.split(X)):
        X_train, X_val = X.iloc[train_idx], X.iloc[val_idx]
        y_train, y_val = y.iloc[train_idx], y.iloc[val_idx]

        # ---- 在训练集上 fit，在验证集上 transform ----
        # 缺失值填充
        imputer = SimpleImputer(strategy='median')
        X_train_filled = pd.DataFrame(
            imputer.fit_transform(X_train),
            columns=X_train.columns
        )
        X_val_filled = pd.DataFrame(
            imputer.transform(X_val),
            columns=X_val.columns
        )

        # 缩放
        scaler = StandardScaler()
        X_train_scaled = scaler.fit_transform(X_train_filled)
        X_val_scaled = scaler.transform(X_val_filled)

        # 特征选择
        from sklearn.feature_selection import SelectKBest, f_regression
        selector = SelectKBest(f_regression, k=50)
        X_train_sel = selector.fit_transform(X_train_scaled, y_train)
        X_val_sel   = selector.transform(X_val_scaled)

        # 训练模型
        from sklearn.linear_model import Ridge
        model = Ridge(alpha=1.0)
        model.fit(X_train_sel, y_train)

        # 评估
        from sklearn.metrics import mean_squared_error
        y_pred = model.predict(X_val_sel)
        score = np.sqrt(mean_squared_error(y_val, y_pred))
        scores.append(score)
        print(f"Fold {fold+1}: RMSE = {score:.4f}")

    print(f"\nAverage RMSE: {np.mean(scores):.4f} ± {np.std(scores):.4f}")
    return scores

# ===== Time Series 专用：TimeSeriesSplit =====
from sklearn.model_selection import TimeSeriesSplit

tscv = TimeSeriesSplit(n_splits=5)
for fold, (train_idx, val_idx) in enumerate(tscv.split(X)):
    # val_idx 总是在 train_idx 之后

    # 确保训练数据不包含未来信息
    print(f"Fold {fold}: train [{train_idx.min()}-{train_idx.max()}], val [{val_idx.min()}-{val_idx.max()}]")

# ===== GroupKFold 防 Group 泄漏 =====
from sklearn.model_selection import GroupKFold

groups = df['user_id']  # 同一个 user 的数据不能同时出现在 train 和 val

for fold, (train_idx, val_idx) in enumerate(gkf.split(X, y, groups)):
    X_train, X_val = X.iloc[train_idx], X.iloc[val_idx]
    # 验证每个 group 只在一个 fold 内
    train_users = set(groups.iloc[train_idx])
    val_users   = set(groups.iloc[val_idx])
    assert len(train_users & val_users) == 0, f"Fold {fold}: 发现有泄漏的 user!"
    print(f"Fold {fold}: train {len(train_users)} users, val {len(val_users)} users, 无泄漏 ✓")
```

## 8. 完整实战 Pipeline 示例

以下是一个可直接用于 Kaggle 竞赛的完整预处理框架：

```python
import pandas as pd
import numpy as np
from sklearn.model_selection import KFold
from sklearn.preprocessing import StandardScaler, RobustScaler, OrdinalEncoder
from sklearn.impute import SimpleImputer
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer

def kaggle_preprocessing_pipeline(train_df, test_df, target_col, categorical_cols, numerical_cols):
    """
    Kaggle 竞赛级完整预处理 Pipeline
    返回可以直接喂给模型的数据
    """
    # 1. 合并 train/test
    train_df['_is_train'] = True
    test_df['_is_train']  = False
    df = pd.concat([train_df.drop(target_col, axis=1), test_df], axis=0, ignore_index=True)

    # 2. 删除高缺失列
    high_miss = [c for c in df.columns if df[c].isnull().mean() > 0.6]
    df.drop(high_miss, axis=1, inplace=True)

    # 3. 常量列删除
    const_cols = [c for c in numerical_cols if df[c].nunique() <= 1]
    df.drop(const_cols, axis=1, inplace=True)

    # 4. 偏态特征做 log1p 变换
    for col in numerical_cols:
        if col in df.columns and df[col].skew() > 1:
            df[col] = np.log1p(df[col].clip(lower=0))

    # 5. 类别特征频率编码
    for col in categorical_cols:
        if col in df.columns:
            freq = df[col].value_counts(normalize=True)
            df[f'{col}_freq'] = df[col].map(freq)

    # 6. 缺失值填充 + 标记
    for col in df.columns:
        if df[col].isnull().any():
            df[f'{col}_ismissing'] = df[col].isnull().astype(int)

    num_imputer = SimpleImputer(strategy='median')
    cat_imputer = SimpleImputer(strategy='most_frequent')

    numeric_cols_final = [c for c in numerical_cols if c in df.columns]
    cat_cols_final = [c for c in categorical_cols if c in df.columns]

    df[numeric_cols_final] = num_imputer.fit_transform(df[numeric_cols_final])
    df[cat_cols_final] = cat_imputer.fit_transform(df[cat_cols_final].astype(str))

    # 7. 拆分回 train/test
    train = df[df['_is_train'] == True].drop('_is_train', axis=1)
    test  = df[df['_is_train'] == False].drop('_is_train', axis=1)

    train[target_col] = train_df[target_col]

    return train, test
```
