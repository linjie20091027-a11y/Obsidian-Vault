# DBSCAN 聚类

## 1. 核心思想

**DBSCAN**（Density-Based Spatial Clustering of Applications with Noise）是一种基于**密度**的聚类算法。

核心思想是：将聚类定义为一个由**高密度区域**经连通性连接而成的最大集合。每个簇由一系列**密度相连**的样本构成，低密度区域的孤立样本被标记为**噪声点**。

与 K-Means 等基于距离/原型的算法不同，DBSCAN 不预设簇的数量，也不假设簇的几何形状，因此能够发现**任意形状的簇**。

---

## 2. 核心概念定义

DBSCAN 基于两个核心参数：**ε**（邻域半径）和 **MinPts**（最小样本数）。

### 2.1 ε-邻域

对于样本点 $p$，其 ε-邻域定义为以 $p$ 为中心、半径 ε 内的所有样本集合：

$$
N_\varepsilon(p) = \{\ q \mid \text{dist}(p, q) \leq \varepsilon\ \}
$$

其中 $\text{dist}(p, q)$ 通常取欧氏距离，但也可推广到其他距离度量。

### 2.2 点的分类

根据 ε-邻域的样本密度，所有点被分为三类：

| 类别 | 定义 | 特征 |
|:---|:---|:---|
| **核心点（Core Point）** | $\|N_\varepsilon(p)\| \geq \text{MinPts}$ | 位于高密度区域，邻域内样本数达到阈值 |
| **边界点（Border Point）** | $\|N_\varepsilon(p)\| < \text{MinPts}$ 且 $p$ 落在某个核心点的 ε-邻域内 | 位于簇的边缘，自身密度不足但附属于核心点 |
| **噪声点（Noise Point）** | 既非核心点也非边界点 | 孤立于低密度区域，不属于任何簇 |

```
示意图（一维简化）：
─────────────────────────────────────────────
   · · · · · · · · · · · · · · · · · · · ·   （· 表示数据点）
   |--core--|   |border|   |noise|
   ε内有≥MinPts  ε内<MinPts   孤立
─────────────────────────────────────────────
```

### 2.3 聚类簇的正式定义

给定参数 ε 和 MinPts，一个**簇 C** 是满足以下两个条件的非空样本子集：

1. **极大性（Maximality）**：$\forall p, q$，若 $p \in C$ 且 $q$ 从 $p$ 密度可达，则 $q \in C$
2. **连通性（Connectivity）**：$\forall p, q \in C$，$p$ 和 $q$ 是密度相连的

---

## 3. 密度关系的形式化定义

DBSCAN 定义了三层递进的密度关系，构建了聚类的理论基础。

### 3.1 密度直达（Directly Density-Reachable）

点 $q$ 从点 $p$ **密度直达**，当且仅当：
1. $p$ 是核心点（$\|N_\varepsilon(p)\| \geq \text{MinPts}$）
2. $q \in N_\varepsilon(p)$（$q$ 在 $p$ 的 ε-邻域内）

即：$q$ 直接被核心点 $p$ 的密度所"覆盖"。

> **注意**：密度直达不满足对称性。若 $q$ 不是核心点，则 $p$ 不一定从 $q$ 密度直达。

### 3.2 密度可达（Density-Reachable）

点 $q$ 从点 $p$ **密度可达**，当且仅当存在一个样本链 $p = p_1, p_2, \ldots, p_n = q$，使得对于每个 $i = 1, 2, \ldots, n-1$，$p_{i+1}$ 从 $p_i$ **密度直达**。

即：存在一条"核心点跳跃路径"从 $p$ 到达 $q$。

> **注意**：密度可达同样不满足对称性，且要求路径中除最后一个点外都必须为核心点。

### 3.3 密度相连（Density-Connected）

点 $p$ 和点 $q$ **密度相连**，当且仅当存在点 $o$，使得 $p$ 和 $q$ 都从 $o$ **密度可达**。

密度相连满足**对称性**——这是将样本划入同一簇的基本依据。

```
关系的层级递进：
────────────────────
密度直达 → 密度可达 → 密度相连
  (局部)    (传导)     (等价关系)
────────────────────
```

## 4. 算法流程

```
算法伪代码：
────────────────────────────────────────────────────────
DBSCAN(X, ε, MinPts):
    标记所有点为 "未访问"
    C = 0                           # 簇编号
    for each 未访问点 p ∈ X:
        标记 p 为 "已访问"
        N = N_ε(p)                  # 获取 p 的 ε-邻域
        if |N| < MinPts:
            标记 p 为 "噪声"         # 暂时标记，后续可能被修正为边界点
        else:
            C = C + 1               # 创建新簇
            expandCluster(p, N, C, ε, MinPts)

expandCluster(p, N, C, ε, MinPts):
    将 p 加入簇 C
    for each 点 q ∈ N:
        if q 未被访问:
            标记 q 为 "已访问"
            N_q = N_ε(q)            # q 的 ε-邻域
            if |N_q| ≥ MinPts:      # q 也是核心点
                N = N ∪ N_q         # 将 q 的邻域加入搜索集合
        if q 不属于任何簇:
            将 q 加入簇 C
────────────────────────────────────────────────────────
```

**关键点**：
- 算法从每个未访问的核心点出发"扩展"簇，将密度相连的点全部纳入
- 对每个核心点，其 ε-邻域内的所有点（包括边界点）都被加入当前簇
- 被标记为噪声的点，若后续发现其落在某个核心点邻域内，会重新归类为边界点
- 每个点最多被访问一次，时间复杂度为 $O(n \cdot \text{cost}(\varepsilon\text{-neighborhood query}))$

## 5. 参数选择

DBSCAN 的效果高度依赖于参数 ε 和 MinPts 的选择。

### 5.1 ε 的选择：K-距离图（K-distance Graph）

对于给定的 MinPts（通常先设定），执行以下步骤：

1. 对每个点 $p$，计算其到第 $k = \text{MinPts} - 1$ 近邻的距离，记为 $\text{k-dist}(p)$
2. 将所有点的 $\text{k-dist}$ 值按降序排列
3. 绘制 K-距离曲线图
4. 选择曲线的**拐点（elbow/knee）**对应的距离值作为 ε

```
K-distance 曲线示意图：
  k-dist
   ↑
   │ \
   │  \
   │   \___ ← 拐点（取此处 ε）
   │       \___________
   └──────────────────→ 排序后的点
    
拐点之前：密集区域的点，k-dist 较小
拐点之后：稀疏/噪声区域的点，k-dist 急剧增大
```

### 5.2 MinPts 的选择

经验规则：
- 一般取 $\text{MinPts} \geq D + 1$，其中 $D$ 为特征维度
- 对于噪声较多的数据集，可适当增大 MinPts
- 对于较大的数据集，可取 $\text{MinPts} = 2 \cdot D$
- **实践中常用范围**：$\text{MinPts} \in [3, 20]$（取决于数据规模和密度）

### 5.3 参数调节策略

1. 先固定 MinPts（如取 $2D$ 或 $4$），用 K-距离图选 ε
2. 逐步调整，观察聚类结果的簇数量和噪声点比例
3. 噪声比例过高 → 增大 ε 或减小 MinPts
4. 簇数过少（大量合并） → 减小 ε 或增大 MinPts

## 6. 优缺点分析

### 6.1 优点

| 优点 | 说明 |
|:---|:---|
| **可发现任意形状的簇** | 基于密度连通性，不受凸/球形假设限制，可处理非凸、缠绕、环状簇 |
| **自动识别噪声** | 将低密度区域的点标记为噪声，无需后处理，对异常值天然鲁棒 |
| **无需预设簇数 K** | 簇的数量由数据本身的密度结构决定，算法自动发现 |
| **只需两个参数** | ε 和 MinPts，且参数有明确的物理含义和选择方法 |
| **对样本输入顺序不敏感** | 不同于 K-Means 受初始质心影响，DBSCAN 的边界点归属可能随顺序微调，但核心结构稳定 |

### 6.2 缺点

| 缺点 | 说明 |
|:---|:---|
| **对 ε 和 MinPts 敏感** | 参数细微变化可能导致聚类结果差异巨大 |
| **密度差异大的数据效果差** | 全局统一的 ε 无法同时适应稀疏簇和密集簇，各簇密度不均匀时表现不佳 |
| **高维数据效果差** | 维数灾难使得距离度量失效，ε-邻域内的"密度"概念丧失意义 |
| **计算开销较大** | 朴素实现需 $O(n^2)$，使用空间索引（如 R*-tree、KD-tree）可降至 $O(n \log n)$ |
| **边界点归属不确定** | 若边界点落在多个核心点邻域内，其归属依赖于遍历顺序 |

## 7. 与 K-Means 的对比

| 维度 | K-Means | DBSCAN |
|:---|:---|:---|
| **原理** | 基于距离（原型） | 基于密度 |
| **簇形状** | 只能发现凸/球形簇 | 可发现任意形状的簇 |
| **预设 K** | 必须预设 | 不需要，自动发现 |
| **噪声处理** | 所有点都被分配（无噪声概念） | 自动标记噪声点 |
| **参数** | K（簇数） | ε（邻域半径）+ MinPts（密度阈值） |
| **对初始值敏感** | 是（质心初始化） | 否（无随机初始化步骤） |
| **可解释性** | 质心可解释 | 核心点可解释 |
| **计算复杂度** | $O(t \cdot K \cdot n)$ | $O(n \log n)$ 到 $O(n^2)$ |
| **离群点** | 敏感（拉偏质心） | 鲁棒（标记为噪声） |
| **适用场景** | 簇大小相近、分布均匀的数据 | 空间数据、地理数据、形状复杂的数据 |

> **选择建议**：若数据簇大致呈球形且大小相近 → K-Means；若簇形状不规则或需检测噪声 → DBSCAN。

## 8. OPTICS 扩展简介

**OPTICS**（Ordering Points To Identify the Clustering Structure）是 DBSCAN 的重要扩展，旨在解决 DBSCAN 对参数 ε 敏感的问题。

### 8.1 核心思想

OPTICS 不直接输出簇分配，而是生成一个**可达性距离图（Reachability Plot）**，该图展示了数据在不同密度层次下的聚类结构。

### 8.2 核心距离与可达距离

**核心距离**（Core Distance）：对于核心点 $p$，使其成为核心点的最小半径：

$$
\text{core-dist}_\varepsilon(p) = \text{MinPts-最近邻距离}, \quad \text{若 } |N_\varepsilon(p)| \geq \text{MinPts}
$$

**可达距离**（Reachability Distance）：从核心点 $p$ 到点 $q$ 的可达距离：

$$
\text{reach-dist}_\varepsilon(p, q) = \max\left(\text{core-dist}_\varepsilon(p),\ \text{dist}(p, q)\right)
$$

### 8.3 可达性图解读

可达性图中**谷底**对应簇，**山峰**对应簇的边界或噪声。通过在不同高度"切割"可达性图，可以获取不同密度层次下的聚类结果——这等效于在不同 ε 下运行 DBSCAN，但只需运行一次。

### 8.4 OPTICS vs DBSCAN

| OPTICS | DBSCAN |
|:---|:---|
| 输出聚类结构的有序表示 | 输出确定的簇标签 |
| 对 ε 不敏感（ε 仅作为计算上限） | 对 ε 高度敏感 |
| 需要从可达性图中提取簇（后处理） | 直接输出簇 |
| 可以发现不同密度的层次化簇结构 | 全局统一密度阈值 |

## 9. 时间复杂度分析

| 实现方式 | 时间复杂度 | 空间复杂度 |
|:---|:---|:---|
| 朴素实现（暴力搜索邻域） | $O(n^2)$ | $O(n)$ |
| 使用 KD-Tree | $O(n \log n)$（低维） | $O(n)$ |
| 使用 R*-tree | $O(n \log n)$ | $O(n)$ |
| 高维数据（无有效索引） | 退化为 $O(n^2)$ | $O(n)$ |

> 对于低维空间数据（$D \leq 10$），KD-Tree 或 R*-tree 可显著加速邻域查询。
