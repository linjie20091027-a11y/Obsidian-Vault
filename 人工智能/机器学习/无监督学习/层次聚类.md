# 层次聚类（Hierarchical Clustering）

## 1. 概述

层次聚类（Hierarchical Clustering）是一种构建**嵌套簇层次结构**的聚类方法。它不预设簇的数量，而是生成一棵**聚类树**（树状图 / Dendrogram），用户可在任意层次"切割"以获取所需数量的簇。

与 K-Means（划分式）和 DBSCAN（密度式）不同，层次聚类产生的是**层级化的聚类结果**，一次计算即可同时获得从 $n$ 个簇到 $1$ 个簇的所有划分方案。

---

## 2. 两种构建策略

### 2.1 凝聚层次聚类（Agglomerative，自底向上）

```
初始化：每个样本作为一个独立的簇（共 n 个簇）
循环：
    找到距离最近的两个簇 Ci, Cj
    将 Ci 和 Cj 合并为一个新簇 C_new = Ci ∪ Cj
    更新距离矩阵（计算 C_new 与其余簇的距离）
直到所有样本合并为一个簇
```

这是最常用的层次聚类方式，绝大多数库实现（如 `sklearn.cluster.AgglomerativeClustering`）采用凝聚策略。

### 2.2 分裂层次聚类（Divisive，自顶向下）

```
初始化：所有样本属于同一个簇（共 1 个簇）
循环：
    选择一个簇
    将该簇分裂为两个子簇（通常用 K-Means K=2 或其他二分策略）
直到每个样本成为一个簇，或达到预设簇数
```

**对比**：

| 维度 | 凝聚聚类 | 分裂聚类 |
|:---|:---|:---|
| 方向 | 自底向上 | 自顶向下 |
| 起始状态 | $n$ 个簇（每个样本一个簇） | $1$ 个簇（全体样本） |
| 分裂决策 | 无需，仅需决定合并哪两个簇 | 需要决定分裂哪个簇、如何分裂 |
| 计算复杂度 | 较高但可控 | 通常更高（需评估所有可能的二分） |
| 实际使用 | 主流，广泛使用 | 较少使用 |

---

## 3. 簇间距离度量（Linkage Criteria）

簇间距离的定义是层次聚类的核心选择，不同的 linkage 方法会产生完全不同的聚类结果。

设簇 $C_i$ 和 $C_j$，其大小分别为 $|C_i|$ 和 $|C_j|$。

### 3.1 单链接（Single Linkage，最近距离）

$$
d(C_i, C_j) = \min_{a \in C_i,\ b \in C_j} \text{dist}(a, b)
$$

- **含义**：两个簇中最接近的两个样本间的距离
- **特点**：容易产生**链式效应（chaining effect）**——只要有一条细线连接，两个簇就会被合并，导致产生细长的簇
- **适用**：能发现非凸形状的簇（类似 DBSCAN 的连通性思想）

### 3.2 全链接（Complete Linkage，最远距离）

$$
d(C_i, C_j) = \max_{a \in C_i,\ b \in C_j} \text{dist}(a, b)
$$

- **含义**：两个簇中最远的两个样本间的距离
- **特点**：倾向于产生**紧凑的球形簇**，对异常值敏感
- **适用**：要求簇内样本两两都相近的场景

### 3.3 平均链接（Average Linkage）

$$
d(C_i, C_j) = \frac{1}{|C_i| \cdot |C_j|} \sum_{a \in C_i} \sum_{b \in C_j} \text{dist}(a, b)
$$

- **含义**：两簇所有样本对距离的均值
- **特点**：单链接与全链接的折中，具有一定的鲁棒性
- **变体**：加权平均链接（WPGMA），考虑子簇大小的影响

### 3.4 沃德方法（Ward's Method）

沃德方法不直接定义簇间距离，而是以**最小化合并后的误差平方和增量**为优化目标。

$$
\Delta(C_i, C_j) = \frac{|C_i| \cdot |C_j|}{|C_i| + |C_j|} \cdot \|\mu_i - \mu_j\|^2
$$

其中 $\mu_i$ 和 $\mu_j$ 分别是簇 $C_i$ 和 $C_j$ 的质心。

- **推导**：合并前后的簇内平方和增量（Lance-Williams 公式的特例）
- **特点**：倾向于合并样本数较少的簇，产生**大小均衡的球形簇**
- **关系**：Ward 方法在最小化方差方面与 K-Means 目标一致，若用于分裂到 K 个簇，结果接近 K-Means

### 3.5 质心链接（Centroid Linkage）

$$
d(C_i, C_j) = \|\mu_i - \mu_j\|
$$

直接以两簇质心的距离作为簇间距离。存在**逆反转（inversion）**问题——合并后新质心可能比合并前更接近某些簇，导致树状图出现交叉。

### 3.6 Lance-Williams 统一公式

上述链接方法可通过 Lance-Williams 递推公式统一表达。若簇 $C_k$ 由 $C_i$ 和 $C_j$ 合并得到，则 $C_k$ 与另一簇 $C_l$ 的距离为：

$$
d(C_k, C_l) = \alpha_i \cdot d(C_i, C_l) + \alpha_j \cdot d(C_j, C_l) + \beta \cdot d(C_i, C_j) + \gamma \cdot |d(C_i, C_l) - d(C_j, C_l)|
$$

不同 linkge 对应的参数：

| Linkage | $\alpha_i$ | $\alpha_j$ | $\beta$ | $\gamma$ |
|:---|:---:|:---:|:---:|:---:|
| **Single** | $1/2$ | $1/2$ | $0$ | $-1/2$ |
| **Complete** | $1/2$ | $1/2$ | $0$ | $+1/2$ |
| **Average** | $\frac{\|C_i\|}{\|C_k\|}$ | $\frac{\|C_j\|}{\|C_k\|}$ | $0$ | $0$ |
| **Ward** | $\frac{\|C_i\|+\|C_l\|}{\|C_k\|+\|C_l\|}$ | $\frac{\|C_j\|+\|C_l\|}{\|C_k\|+\|C_l\|}$ | $\frac{-\|C_l\|}{\|C_k\|+\|C_l\|}$ | $0$ |
| **Centroid** | $\frac{\|C_i\|}{\|C_k\|}$ | $\frac{\|C_j\|}{\|C_k\|}$ | $-\frac{\|C_i\|\|C_j\|}{\|C_k\|^2}$ | $0$ |

## 4. 算法复杂度

### 4.1 朴素实现（Naive）

- 距离矩阵构建：$O(n^2 \cdot D)$（$D$ 为维度）
- 每步合并扫描距离矩阵找最小值：$O(n^2)$
- 共需 $n-1$ 步合并
- 总复杂度：**$O(n^3)$**（不可接受）

### 4.2 使用优先队列（堆）优化

- 初始构建距离矩阵：$O(n^2)$
- 维护一个最小堆存储最邻近簇对
- 每次合并后更新受影响的距离：$O(n \log n)$
- 总复杂度：**$O(n^2 \log n)$**

### 4.3 实际可用方法

| 方法 | 时间复杂度 | 空间复杂度 | 说明 |
|:---|:---|:---|:---|
| 朴素 | $O(n^3)$ | $O(n^2)$ | 仅限教学 |
| 堆优化 | $O(n^2 \log n)$ | $O(n^2)$ | 一般实现 |
| SLINK（单链接专用） | $O(n^2)$ | $O(n)$ | 利用单链接特殊性质 |
| CLINK（全链接专用） | $O(n^2)$ | $O(n)$ | 利用全链接特殊性质 |
| BIRCH (CF-Tree) | $O(n)$ | 可控 | 大规模数据的近似方案 |

> 对于百万级数据，可先用 BIRCH 等算法压缩数据，再对压缩后的子簇进行层次聚类。

## 5. 树状图（Dendrogram）解读

树状图是层次聚类结果的可视化表示，形如倒置的树：

```
       ┌──────────────────────────────┐
       │           Root               │  ← 所有样本（1个簇）
       │      ┌─────────┴─────────┐   │
       │   ┌──┴──┐            ┌──┴──┐ │
       │ ┌─┴─┐ ┌─┴─┐      ┌─┴─┐ ┌─┴─┐│
       │ A  B  C  D  E      F  G  H  I│  ← 叶子节点（每个样本一个簇）
```

**解读要点**：

| 要素 | 含义 |
|:---|:---|
| **纵轴高度** | 合并时两个簇的距离/差异度；高度越大，被合并的簇差异越大 |
| **横轴排列** | 叶子节点的排列顺序（可调整，不影响聚类结构）；相邻叶子不一定属于同一簇 |
| **水平切割线** | 在某一高度画水平线，与树的垂直分支相交的次数 = 该高度下的簇数 |
| **切割线位置** | 选择分支较长（合并距离跳跃大）的位置进行切割，可获得自然的聚类划分 |

### 切割原则

最佳切割位置应在树状图中"垂直分支最长"的地方：

- 在某一高度附近，合并距离出现**跃增（jump）**，说明再往上合并是"强行"将不同簇合并在一起
- 切割线应位于合并距离跳跃的中间位置

## 6. 优缺点

### 6.1 优点

| 优点 | 说明 |
|:---|:---|
| **无需预设簇数 K** | 可从树状图任意高度切割获取所需簇数，灵活性强 |
| **层次化结构** | 一次计算即可获得所有层次的聚类结果，适合探索性数据分析 |
| **可解释性强** | 树状图直观展示聚类过程和样本间的相似关系 |
| **可处理任意距离** | 只要样本间距离可定义，即可进行层次聚类（不限于欧氏距离） |
| **确定性** | 无随机初始化，结果完全由数据和 linkage 方法决定，可复现 |
| **配合不同 linkage 灵活** | 根据业务需求选择合适的 linkage 准则 |

### 6.2 缺点

| 缺点 | 说明 |
|:---|:---|
| **计算复杂度高** | 一般 $O(n^2 \log n)$，难以直接处理大规模数据 |
| **空间复杂度高** | 需要 $O(n^2)$ 存储距离矩阵 |
| **合并决策不可逆** | 一旦合并（或分裂），错误无法在后续步骤纠正 |
| **对噪声和异常值敏感** | 特别是 Single-linkage 对噪声敏感，Ward 对异常值敏感 |
| **linkage 选择影响大** | 不同的 linkage 可能产生完全不同的聚类结果，需要领域知识辅助选择 |
| **不适合高维数据** | 维数灾难导致距离度量失效，层次结构失真 |

---

## 7. Linkage 方法选择指南

| 场景 | 推荐 Linkage | 原因 |
|:---|:---|:---|
| 期望球形、大小均衡的簇 | **Ward** | 最小化方差增量，结果与 K-Means 目标一致 |
| 数据中有噪声 | **Average** | 比 Single 和 Complete 更鲁棒 |
| 需要发现任意形状的簇 | **Single** | 链式效应可以发现细长的非凸簇 |
| 严格要求簇内紧密度 | **Complete** | 确保簇内所有点两两之间都不太远 |
| 不确定，寻求通用方案 | **Ward** 或 **Average** | 大多数情况下表现稳定 |

## 8. 与 K-Means 和 DBSCAN 的综合对比

| 维度 | K-Means | DBSCAN | 层次聚类 |
|:---|:---|:---|:---|
| **预设簇数** | 必须预设 K | 自动发现 | 不预设，从树状图切割 |
| **簇形状** | 凸/球形 | 任意形状 | 取决于 linkage |
| **噪声处理** | 全部分配 | 自动标记噪声 | 全部分配（无噪声概念） |
| **可复现性** | 受初始值影响（需多次运行） | 确定性（边界点顺序影响微小） | 确定性 |
| **计算复杂度** | $O(tKn)$ | $O(n \log n)$ ~ $O(n^2)$ | $O(n^2 \log n)$ |
| **大数据支持** | 好（Mini-Batch 变体） | 中等（需空间索引） | 差（需距离矩阵） |
| **层级信息** | 无 | 无（OPTICS 变体有） | 有（核心输出） |
| **适用场景** | 快速划分、已知 K 的大规模数据 | 空间数据、任意形状簇、带噪声数据 | 探索性分析、小到中等规模、需层次结构 |

## 9. Lance-Williams 公式的几何直觉

结合 3.6 节的统一公式，不同 linkage 的核心差异在于 $\gamma$ 参数（是否考虑现有距离差）：

- $\gamma = -1/2$（Single）：新簇距离取两者中的最小值——**乐观合并**，只看最近点
- $\gamma = +1/2$（Complete）：新簇距离取两者中的最大值——**悲观合并**，看最远点
- $\gamma = 0$（Average / Ward）：取**折中**值

这种差异从根本上决定了树状图的形状和最终聚类结果的特性。
