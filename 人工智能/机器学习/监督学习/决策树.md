# 决策树（Decision Tree）

## 一、核心思想

决策树的核心思想是：**递归地将特征空间划分为子区域，使得每个子区域内的样本尽可能属于同一类别（分类）或输出值尽可能接近（回归）**。

直观理解：
- 从根节点开始，选择一个最优特征进行划分
- 每个内部节点代表一次特征测试，每个分支代表测试结果，每个叶节点代表一个决策
- 划分过程持续直到满足停止条件（纯节点、最大深度、最小样本数等）
- 决策树模拟了人类"if-then"的决策过程，天然可解释

数学上，决策树的生成是一个**递归选择最优划分特征**的过程，不同的特征选择准则对应不同的算法。

## 二、特征选择准则（详细公式推导）

### 2.1 信息熵与信息增益（ID3 算法）

#### 信息熵定义

设数据集 $D$ 中第 $k$ 类样本的比例为 $p_k$（$k = 1, 2, \dots, K$），则 $D$ 的**信息熵**为：

$$H(D) = -\sum_{k=1}^{K} p_k \log_2 p_k$$

信息熵衡量数据集的不确定性：
- 当样本均匀分布在各类时，$H(D)$ 最大（$\log_2 K$）
- 当所有样本同属一类时，$H(D) = 0$（纯节点）

#### 条件熵

设特征 $A$ 有 $V$ 个可能取值 $\{a_1, a_2, \dots, a_V\}$，用 $A$ 对 $D$ 划分得到 $V$ 个子集 $D_1, D_2, \dots, D_V$。划分后的**条件熵**为：

$$H(D|A) = \sum_{v=1}^{V} \frac{|D_v|}{|D|} H(D_v)$$

其中 $|D_v|/|D|$ 表示第 $v$ 个分支的权重。

#### 信息增益

**信息增益（Information Gain）**定义为划分前后熵的减少量：

$$\text{Gain}(D, A) = H(D) - H(D|A) = H(D) - \sum_{v=1}^{V} \frac{|D_v|}{|D|} H(D_v)$$

信息增益越大，说明使用特征 $A$ 划分后不确定性降低越多。

**ID3 算法的核心**：在每个节点选择信息增益最大的特征进行划分。

#### 信息增益的缺陷

信息增益偏好**取值数目较多**的特征。极端情况：若每个样本在某特征上取值都不同，则每个子集 $D_v$ 只含一个样本，$H(D_v) = 0$，信息增益达到最大值，但这种划分泛化能力极差。

---

### 2.2 信息增益率（C4.5 算法）

为解决信息增益偏向多值特征的问题，C4.5 引入**信息增益率（Gain Ratio）**：

$$\text{Gain\_ratio}(D, A) = \frac{\text{Gain}(D, A)}{\text{SplitInfo}_A(D)}$$

其中**分裂信息**（Split Information）衡量特征 $A$ 本身的熵：

$$\text{SplitInfo}_A(D) = H_A(D) = -\sum_{v=1}^{V} \frac{|D_v|}{|D|} \log_2 \frac{|D_v|}{|D|}$$

**原理**：
- 若特征 $A$ 取值非常多（如 ID 特征），$\text{SplitInfo}_A(D)$ 会很大，从而压缩信息增益率
- 若特征 $A$ 取值很少（如二值特征），$\text{SplitInfo}_A(D)$ 很小，增益率则相对较大
- C4.5 实际使用启发式策略：先从候选特征中选出信息增益高于平均水平的，再从中选择增益率最大的

---

### 2.3 基尼指数（CART 分类树）

**基尼值（Gini Index）**衡量数据集的不纯度：

$$\text{Gini}(D) = \sum_{k=1}^{K} p_k (1 - p_k) = 1 - \sum_{k=1}^{K} p_k^2$$

其中 $p_k$ 为第 $k$ 类样本的比例。

**基尼值的性质**：
- 当所有样本同属一类时，$\text{Gini}(D) = 0$（最纯）
- 当样本均匀分布在各类时，$\text{Gini}(D) = 1 - 1/K$（最大不纯）
- 基尼值可理解为：从 $D$ 中随机抽取两个样本，它们类别不一致的概率

**特征 $A$ 的基尼指数**（划分后的加权基尼值）：

$$\text{Gini\_index}(D, A) = \sum_{v=1}^{V} \frac{|D_v|}{|D|} \text{Gini}(D_v)$$

**CART 算法**选择基尼指数最小的特征进行划分（基尼指数越小，划分后纯度越高）。

---

### 2.4 三种准则对比总结

| 准则 | 公式 | 算法 | 特点 |
|------|------|------|------|
| 信息增益 | $\text{Gain} = H(D) - \sum \frac{|D_v|}{|D|} H(D_v)$ | ID3 | 偏向多值特征 |
| 信息增益率 | $\text{Gain\_ratio} = \frac{\text{Gain}}{\text{SplitInfo}}$ | C4.5 | 校正多值偏向 |
| 基尼指数 | $\text{Gini\_index} = \sum \frac{|D_v|}{|D|} \text{Gini}(D_v)$ | CART | 计算高效，无需对数 |

**实际应用中**：三种准则在大多数数据集上结果相似，信息增益和基尼指数在效果上差异不大，但基尼指数因无需对数计算而更高效。

## 三、三种经典算法对比

### 3.1 ID3（Iterative Dichotomiser 3）

| 项目 | 说明 |
|------|------|
| **提出者** | Quinlan (1986) |
| **特征选择** | 信息增益（Information Gain） |
| **树结构** | **多叉树**（每个特征取值对应一个分支） |
| **特征类型** | 仅支持离散（分类）特征 |
| **剪枝** | 无内置剪枝 |
| **缺失值处理** | 不支持 |
| **主要缺陷** | 偏向多值特征；容易过拟合；无法处理连续值 |

### 3.2 C4.5

| 项目 | 说明 |
|------|------|
| **提出者** | Quinlan (1993) |
| **特征选择** | 信息增益率（Gain Ratio） |
| **树结构** | **多叉树** |
| **特征类型** | 离散 + 连续（自动二分离散化） |
| **剪枝** | 后剪枝（悲观剪枝 Pessimistic Pruning） |
| **缺失值处理** | 支持（概率权重法） |
| **改进** | C5.0 / See5（商业化版本，提升速度和内存效率） |

**连续值处理**：C4.5 将连续属性离散化——排序后取相邻值的中点作为候选分裂点，选择信息增益率最大的分裂点。

**悲观剪枝（Pessimistic Pruning）**：基于训练集的错误率进行统计修正，使用二项分布的置信上限来估计泛化错误率。

### 3.3 CART（Classification and Regression Tree）

| 项目 | 说明 |
|------|------|
| **提出者** | Breiman et al. (1984) |
| **特征选择** | 分类：基尼指数；回归：最小二乘偏差 |
| **树结构** | **二叉树**（每次划分仅产生两个分支） |
| **剪枝** | 后剪枝（代价复杂度剪枝 CCP） |
| **特有功能** | 同时支持分类和回归 |

**CART 的二叉树特点**：
- 即使特征是离散多值的，CART 也会将其二值化（如 {$a_1, a_3$} vs 其余）
- 这使得 CART 的树结构更灵活，避免了 ID3/C4.5 中多值特征分支过细的问题

## 四、回归树（Regression Tree）

### 4.1 核心思想

回归树的目标是预测**连续值**。每个叶节点输出该节点内样本目标值的均值。

### 4.2 划分准则：最小二乘偏差

设划分后两个子区域 $R_1$ 和 $R_2$（CART 是二叉树），在每个区域内寻找最优输出值 $c_1, c_2$ 使得平方误差最小：

$$\min_{j, s} \left[ \min_{c_1} \sum_{x_i \in R_1(j,s)} (y_i - c_1)^2 + \min_{c_2} \sum_{x_i \in R_2(j,s)} (y_i - c_2)^2 \right]$$

其中：
- $j$ 为划分特征的下标，$s$ 为划分点的值
- $R_1(j, s) = \{x \mid x^{(j)} \leq s\}$，$R_2(j, s) = \{x \mid x^{(j)} > s\}$

最优输出分别为区域内的均值：
$$c_1 = \frac{1}{|R_1|} \sum_{x_i \in R_1} y_i, \quad c_2 = \frac{1}{|R_2|} \sum_{x_i \in R_2} y_i$$

### 4.3 完整划分过程

对每个特征 $j$ 和每个可能的划分点 $s$：
1. 将数据分为左右两部分
2. 计算两部分各自的均值作为预测值
3. 计算两部分各自的平方误差之和
4. 选择使总平方误差最小的 $(j, s)$
5. 递归地对两个子区域重复上述过程

## 五、剪枝（Pruning）

决策树容易过度拟合训练数据。剪枝是防止过拟合的关键技术。

### 5.1 预剪枝（Pre-pruning）

在树生长过程中提前停止分裂。常见策略：

| 策略 | 说明 |
|------|------|
| **最大深度（max_depth）** | 限制树的最大层数 |
| **最小样本分裂数（min_samples_split）** | 节点样本数小于阈值时不分裂 |
| **最小叶节点样本数（min_samples_leaf）** | 分裂后叶节点样本数小于阈值则不分裂 |
| **最小不纯度减少量（min_impurity_decrease）** | 分裂带来的不纯度降低小于阈值时不分裂 |
| **最大叶节点数（max_leaf_nodes）** | 限制叶节点总数 |

**优点**：简单高效，训练时间短
**缺点**：难以确定最佳停止时机（太早则欠拟合，太晚则过拟合）；某些看似无用的分裂可能为后续关键分裂创造条件（视野盲区效应）

### 5.2 后剪枝（Post-pruning）

先生成一棵完全生长的树，再自底向上修剪。

#### CCP 代价复杂度剪枝（Cost-Complexity Pruning）

定义树的**代价复杂度**：

$$R_\alpha(T) = R(T) + \alpha |T|$$

其中：
- $R(T)$ 为树 $T$ 在训练集上的误差（分类错误率或回归均方误差）
- $|T|$ 为树的叶节点数量
- $\alpha \geq 0$ 为复杂度惩罚系数

**剪枝过程**：
1. 对每个内部节点，计算将其子树替换为叶节点后的代价复杂度变化
2. 选择使 $R_\alpha(T)$ 降低最多的节点进行剪枝
3. 逐步增大 $\alpha$，得到一系列嵌套子树序列 $T_0 \supset T_1 \supset \dots \supset T_{root}$
4. 通过交叉验证选择最优 $\alpha$（最佳复杂度-精度权衡）

**C4.5 的悲观剪枝**：
- 不需要验证集，仅使用训练集
- 对每个节点的错误率进行连续性校正（二项分布的正态近似）
- 若剪枝后错误率的置信上界低于剪枝前，则执行剪枝

## 六、连续值与缺失值处理

### 6.1 连续值处理

- **二分法（C4.5 / CART）**：将连续特征排序，取相邻值的中点作为候选划分点
  - 若有 $m$ 个不同取值，则有 $m-1$ 个候选划分点
  - 对每个候选点计算划分准则，选择最优的
  - 同一连续特征可以在不同节点多次使用

### 6.2 缺失值处理

**C4.5 的方法**：
1. **划分特征选择**：仅使用该特征不缺失的样本计算信息增益，再乘以"不缺失比例"进行修正
2. **样本分配**：缺失特征的样本以概率权重分配到各分支（权重 = 各分支样本比例），最终叶节点以加权方式投票

**CART 的方法**：使用**替代分裂（Surrogate Split）**——对于每个最优分裂，寻找与其划分结果最相似的其他特征作为替代分裂，缺失值时使用替代分裂。

## 七、优缺点

### 优点

| 优点 | 说明 |
|------|------|
| **可解释性强** | 树结构可视化直观，规则可提取为 if-then 形式，便于向非技术人员解释 |
| **无需特征缩放** | 不依赖距离度量，不需要标准化/归一化 |
| **可处理混合类型** | 同时处理离散和连续特征 |
| **特征选择能力** | 自动进行特征选择，可输出特征重要性排序 |
| **鲁棒性** | 对异常值相对鲁棒（不同于基于距离的模型） |
| **数据预处理少** | 不需要独热编码、标准化等 |
| **非线性建模** | 天然处理非线性关系 |
| **计算开销可控** | 预测时间复杂度 $O(\log N)$，训练可并行 |

### 缺点

| 缺点 | 说明 |
|------|------|
| **易过拟合** | 不剪枝时容易完全拟合训练数据，泛化能力差 |
| **不稳定（高方差）** | 数据的微小变化可能导致完全不同的树结构 |
| **贪心优化** | 每次选择当前最优分裂，不能保证全局最优 |
| **倾斜数据敏感** | 类别不平衡时信息增益偏向多数类 |
| **难以捕捉异或等关系** | 轴平行划分无法有效表达某些复杂边界（需多层划分） |
| **不支持在线学习** | 新增样本通常需要重新训练 |
| **回归的阶梯状输出** | 回归树输出分段常数，无法平滑插值 |

## 八、与其他算法的联系

| 对比算法 | 联系与区别 |
|----------|-----------|
| **随机森林** | Bagging + 随机特征子集 + 决策树；降低方差，提升泛化能力，但牺牲可解释性 |
| **梯度提升树（GBDT/XGBoost/LightGBM/CatBoost）** | Boosting 框架 + 决策树；每棵树拟合前一轮的残差，将弱学习器集成为强学习器 |
| **逻辑回归** | 线性模型 vs 非线性模型；LR 需要特征工程，决策树自动发现特征交互 |
| **SVM** | SVM 基于间隔最大化，需核函数处理非线性；决策树天然处理非线性；SVM 对高维稀疏数据更好 |
| **kNN** | kNN 懒惰学习，依赖距离度量；决策树 eager 学习，无需距离度量 |
| **神经网络** | 决策树可解释，NN 黑盒；NN 可学习复杂函数，决策树在表格数据上往往更优 |
| **朴素贝叶斯** | 朴素贝叶斯基于条件独立假设（常不成立），决策树自动捕获特征依赖 |

## 九、集成学习中的决策树

决策树是现代集成学习的核心基学习器。几乎所有主流提升和装袋算法都以决策树为基：

### 9.1 Bagging 类

- **随机森林（Random Forest）**：多棵 CART 树 + Bootstrap 采样 + 随机特征子集 + 投票/平均

### 9.2 Boosting 类

- **AdaBoost**：自适应提升，基于加权错误率调整样本权重
- **GBDT（梯度提升决策树）**：每棵树拟合损失函数的负梯度
- **XGBoost**：GBDT 的高效实现，加入正则化、列采样、近似分裂点等
- **LightGBM**：基于直方图的梯度提升，GOSS 采样 + EFB 特征捆绑，适合大规模数据
- **CatBoost**：原生支持类别特征，使用 Ordered Boosting 减少预测偏移

## 十一、核心公式速查

### 信息熵（香农熵）

$$H(D) = -\sum_{k=1}^{K} p_k \log_2 p_k$$

### 信息增益

$$\text{Gain}(D, A) = H(D) - \sum_{v=1}^{V} \frac{|D_v|}{|D|} H(D_v)$$

### 信息增益率

$$\text{Gain\_ratio}(D, A) = \frac{\text{Gain}(D, A)}{H_A(D)}$$

$$H_A(D) = -\sum_{v=1}^{V} \frac{|D_v|}{|D|} \log_2 \frac{|D_v|}{|D|}$$

### 基尼指数

$$\text{Gini}(D) = 1 - \sum_{k=1}^{K} p_k^2$$

$$\text{Gini\_index}(D, A) = \sum_{v=1}^{V} \frac{|D_v|}{|D|} \text{Gini}(D_v)$$

### 回归树平方误差

$$\min_{j,s} \left[ \sum_{x_i \in R_1(j,s)} (y_i - \bar{y}_{R_1})^2 + \sum_{x_i \in R_2(j,s)} (y_i - \bar{y}_{R_2})^2 \right]$$

### 代价复杂度剪枝

$$R_\alpha(T) = R(T) + \alpha |T|$$

### 特征重要性（CART）

$$\text{Importance}(A) = \sum_{t: \text{split on }A} \frac{|D_t|}{|D|} \cdot \Delta\text{Impurity}(t)$$

其中 $\Delta\text{Impurity}(t)$ 为节点 $t$ 处分裂前后的不纯度减少量。
