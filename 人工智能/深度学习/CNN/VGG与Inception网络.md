# VGG 与 Inception 网络

---

## 第一部分：VGG 网络

### 1. 概述

VGG（Visual Geometry Group）由牛津大学 VGG 组于 2014 年提出，在 ILSVRC 2014 定位任务第 1 名、分类任务第 2 名。其核心贡献在于**系统的架构设计实验**，证明了网络深度和使用统一的小卷积核对性能有显著提升。

### 2. 核心设计原则：小卷积核堆叠

VGG 的一个关键洞见是：**用多个小卷积核的堆叠替代一个大卷积核**。

#### 2.1 感受野等价性

两个 $3 \times 3$ 卷积层堆叠，等效感受野 = 一个 $5 \times 5$ 卷积层。

推导：第一层 Conv(3×3) 输出上每个点具有 $3 \times 3$ 感受野；第二层 Conv(3×3) 对该输出再卷积，最终每个点的感受野为 $5 \times 5$：

$$RF = 1 + (3-1) \times 1 + (3-1) \times 1 = 5$$

三个 $3 \times 3$ 卷积堆叠，等效感受野 = $7 \times 7$：

$$RF = 1 + (3-1) + (3-1) + (3-1) = 7$$

更通用的结论：$n$ 个 $3 \times 3$ 卷积堆叠的等效感受野为 $(2n+1) \times (2n+1)$。

#### 2.2 优势分析

假设输入输出通道数均为 $C$，比较单个 $5 \times 5$ 卷积与两个 $3 \times 3$ 卷积：

| 方案 | 参数量 | 计算量 |
|------|--------|--------|
| 1× 5×5 Conv | $5^2 \times C^2 = 25C^2$ | $HWC^2 \times 25$ |
| 2× 3×3 Conv | $2 \times 3^2 \times C^2 = 18C^2$ | $HWC^2 \times 18$ |

**两个 $3 \times 3$ 卷积的参数量仅为 $5 \times 5$ 的 72%**，同时：
1. **更多的非线性**：每层后跟 ReLU，增加了决策函数的判别力
2. **更少的参数**：降低了过拟合风险
3. **保持相同的感受野**

类似地，三个 $3 \times 3$ 卷积（$27C^2$）vs 一个 $7 \times 7$ 卷积（$49C^2$），节省约 45% 参数。

### 3. VGG-16 详细结构

```
Input (224×224×3)
│
├─ Block1
│  ├─ Conv3×3, 64, s=1, p=1 → ReLU
│  └─ Conv3×3, 64, s=1, p=1 → ReLU
│  └─ MaxPool 2×2, s=2 → Output: 112×112×64
│
├─ Block2
│  ├─ Conv3×3, 128, s=1, p=1 → ReLU
│  └─ Conv3×3, 128, s=1, p=1 → ReLU
│  └─ MaxPool 2×2, s=2 → Output: 56×56×128
│
├─ Block3
│  ├─ Conv3×3, 256, s=1, p=1 → ReLU
│  ├─ Conv3×3, 256, s=1, p=1 → ReLU
│  ├─ Conv3×3, 256, s=1, p=1 → ReLU (仅VGG-19在此有第3个Conv)
│  └─ MaxPool 2×2, s=2 → Output: 28×28×256
│
├─ Block4
│  ├─ Conv3×3, 512, s=1, p=1 → ReLU
│  ├─ Conv3×3, 512, s=1, p=1 → ReLU
│  ├─ Conv3×3, 512, s=1, p=1 → ReLU (仅VGG-19在此有第3个Conv)
│  └─ MaxPool 2×2, s=2 → Output: 14×14×512
│
├─ Block5
│  ├─ Conv3×3, 512, s=1, p=1 → ReLU
│  ├─ Conv3×3, 512, s=1, p=1 → ReLU
│  ├─ Conv3×3, 512, s=1, p=1 → ReLU (仅VGG-19在此有第3个Conv)
│  └─ MaxPool 2×2, s=2 → Output: 7×7×512
│
├─ FC1: 7×7×512 = 25088 → 4096, ReLU, Dropout(0.5)
├─ FC2: 4096 → 4096, ReLU, Dropout(0.5)
├─ FC3: 4096 → 1000, Softmax
```

### 4. VGG-16 vs VGG-19 对比

| 特性 | VGG-16 | VGG-19 |
|------|--------|--------|
| 卷积层数 | 13 | 16 |
| 全连接层数 | 3 | 3 |
| **总层数** | **16** | **19** |
| 参数量 | ~138M | ~144M |
| 卷积块3的Conv数 | 2 | 3 |
| 卷积块4的Conv数 | 2 | 3 |
| 卷积块5的Conv数 | 2 | 3 |

VGG-19 相比 VGG-16 在 Conv3/4/5 各增加了一个卷积层，参数量仅增加 ~6M。

### 5. VGG 架构规律

| 规律 | 说明 |
|------|------|
| 全部使用 3×3 卷积 | 最小能捕获方向信息的卷积核尺寸 |
| 通道数逐层翻倍 | 64→128→256→512→512，补偿空间缩小的信息损失 |
| 空间分辨率逐层减半 | 每个 MaxPool 将尺寸减半（224→112→56→28→14→7） |
| 单层参数量分布 | 前几层参数量小（通道少），后几层参数量巨大（512×512） |

### 6. VGG 的不足

- **参数量过大**：VGG-16 约 138M 参数，其中全连接层贡献了绝大部分（约 120M），导致存储和部署困难
- **计算量大**：约 15.5 GFLOPs，特别在全连接层
- **训练慢**：需要大量 GPU 内存
- **无跳跃连接/残差**：相比 ResNet 收敛更慢，精度上限也更低

## 第二部分：Inception 网络

### 7. GoogLeNet / Inception v1（2014）

#### 7.1 动机

GoogLeNet（2014 ILSVRC 分类冠军）的核心问题是：**视觉信息在不同图像中以不同尺度出现**——如何为每一层选择正确的卷积核大小？

传统做法是在不同层使用不同大小的卷积核（手动设计），Inception 模块的答案是：**在同一层并行使用多个尺度的卷积核**，让网络自己学习选择。

#### 7.2 朴素 Inception 模块

```
         输入 (Previous Layer)
              │
    ┌─────────┼─────────┬───────────┐
    ▼         ▼         ▼           ▼
 Conv 1×1   Conv 3×3  Conv 5×5   MaxPool 3×3
    │         │         │           │
    └─────────┴─────────┴───────────┘
              │
       通道拼接 (Filter Concatenation)
```

四个并行的分支分别提取不同尺度的特征，最后在通道维度上拼接。

**问题**：计算量巨大，尤其是 $5 \times 5$ 卷积分支。

#### 7.3 带降维的 Inception 模块

在 $3 \times 3$ 和 $5 \times 5$ 卷积前、以及池化后添加 $1 \times 1$ 卷积进行**降维**（Bottleneck Layer）：

```
              输入 (Previous Layer)
                   │
    ┌──────────────┼──────────────┬──────────────────┐
    ▼              ▼              ▼                  ▼
 Conv 1×1      Conv 1×1      MaxPool 3×3, s=1     Conv 1×1
    │              │              │                  │
    │          Conv 3×3      Conv 1×1               │
    │              │              │                  │
    │              │          Conv 5×5               │
    │              │              │                  │
    └──────────────┴──────────────┴──────────────────┘
                   │
            通道拼接 (Filter Concatenation)
```

**计算量分析**（以典型输入 28×28×256 为例）：

没有 $1 \times 1$ 降维时，直接使用 64 个 $5 \times 5$ 卷积：
$$\text{Ops} = 28 \times 28 \times 5 \times 5 \times 256 \times 64 \approx 320\text{M}$$

加入 $1 \times 1$ 降维至 32 通道后再用 $5 \times 5$ 卷积：
$$\text{Ops} = 28^2 \times 1^2 \times 256 \times 32 + 28^2 \times 5^2 \times 32 \times 64 \approx 6.4\text{M} + 40\text{M} \approx 46\text{M}$$

**计算量降低约 85%**。

#### 7.4 GoogLeNet 整体架构

| 模块 | 输出尺寸 | 说明 |
|------|----------|------|
| Conv 7×7 | 112×112×64 | 初始大卷积，stride 2 |
| MaxPool 3×3 | 56×56×64 | stride 2 |
| Conv 1×1 → Conv 3×3 | 56×56×192 | 降维 + 卷积 |
| MaxPool 3×3 | 28×28×192 | stride 2 |
| Inception (3a, 3b) | 28×28×256, 28×28×480 | 两个 Inception 模块 |
| MaxPool 3×3 | 14×14×480 | stride 2 |
| Inception (4a~4e) | 14×14×512/512/512/528/832 | 5 个 Inception 模块 |
| MaxPool 3×3 | 7×7×832 | stride 2 |
| Inception (5a, 5b) | 7×7×832, 7×7×1024 | 2 个 Inception 模块 |
| AvgPool 7×7 | 1×1×1024 | 全局平均池化 |
| FC + Softmax | 1000 | 分类输出 |

**辅助分类器**（Auxiliary Classifier）：

在 Inception (4a) 和 Inception (4d) 之后各插入一个辅助分类器：
```
AvgPool 5×5 → Conv 1×1 (128) → FC (1024) → FC (1000) → Softmax
```

辅助分类器在训练时以较小权重（×0.3）加入总损失，提供额外的梯度信号，起到**正则化 + 缓解梯度消失**的作用。推理时移除。

#### 7.5 GoogLeNet 参数量

约 6.8M 参数（远小于 VGG-16 的 138M），但性能不输——证明了**效率与精度可以兼得**。

### 8. Inception v2 和 v3（2015）

#### 8.1 Inception v2 改进

**大卷积核分解（Factorization）**：

将 $5 \times 5$ 卷积替换为两个堆叠的 $3 \times 3$ 卷积：

```
    输入
      │
 Conv 3×3 → ReLU → Conv 3×3 → ReLU
```

等效感受野为 $5 \times 5$，但参数量从 $25C^2$ 降至 $18C^2$（减少 28%），且增加了一个非线性层。

新的 Inception 模块（Inception A）：
```
输入 → [1×1 Conv] [1×1→3×3 Conv] [1×1→3×3→3×3 Conv] [Pool→1×1 Conv] → 拼接
```

#### 8.2 非对称卷积分解

将 $n \times n$ 卷积分解为 $n \times 1$ 后接 $1 \times n$：

$$\text{Conv }(n \times n) \approx \text{Conv }(n \times 1) \rightarrow \text{Conv }(1 \times n)$$

例如将 $3 \times 3$ 分解为 $3 \times 1$ 加 $1 \times 3$：
- 原参数量：$9C^2$
- 分解后参数量：$(3 \times 1 + 1 \times 3) \times C^2 = 6C^2$

**减少 33% 参数，且计算效率更高。**

Inception B 模块（$n=7$ 可分解）：
```
输入 → [1×1 Conv] [1×1→1×7→7×1 Conv] [1×1→1×7→7×1→1×7→7×1 Conv] [Pool→1×1] → 拼接
```

#### 8.3 高效栅格缩减（Grid Reduction）

在下采样时，先用池化缩小空间尺寸，同时用卷积增加通道数，避免直接进行 stride-2 卷积/池化造成的信息瓶颈。

并行使用两个分支：卷积（stride 2）和池化（stride 2），结果在通道维度拼接，平滑过渡到新的分辨率。

#### 8.4 Inception v3 改进

- **Label Smoothing Regularization (LSR)**：对标签添加均匀噪声，防止极端概率输出，提升泛化能力
- **RMSProp 优化器**
- **Batch Normalization**（Inception v1/v2 中使用，v3 继续沿用）
- **更细致的 $7 \times 7$ 分解**：将 Inception v2 中首个 $7 \times 7$ 卷积分解为三个 $3 \times 3$ 卷积

### 9. Inception v4 和 Inception-ResNet（2016）

#### 9.1 Inception v4

- 架构更加统一整洁
- 模块设计更加规则化
- 引入专门的 Stem 模块（初始输入处理部分），替代简单的 Conv + Pool

#### 9.2 Inception-ResNet v1 / v2

将 ResNet 的**残差连接**融入 Inception 架构：

$$\mathbf{y} = \text{Inception}(\mathbf{x}) + \mathbf{x}$$

- **Inception-ResNet v1**：计算量与 Inception v3 接近
- **Inception-ResNet v2**：计算量与 Inception v4 接近，性能更优

**特点**：
- 残差连接加速训练收敛
- Inception 模块内部结构保持多尺度并行
- 引入了缩放因子（约 0.1~0.3）乘以残差分支，稳定训练（防止网络在早期因残差过大而"死掉"）

## 第三部分：多模型对比

### 10. 参数量与计算量对比

| 模型 | 年份 | 参数量 | FLOPs | Top-5 Error | 层数 |
|------|------|--------|-------|-------------|------|
| AlexNet | 2012 | 60M | 1.5G | 16.4% | 8 |
| VGG-16 | 2014 | 138M | 15.5G | 7.3% | 16 |
| VGG-19 | 2014 | 144M | 19.6G | 7.1% | 19 |
| GoogLeNet (Inception v1) | 2014 | 6.8M | 1.5G | 6.7% | 22 |
| Inception v3 | 2015 | 23.8M | 5.7G | 3.6% | 48 |
| ResNet-50 | 2015 | 25.6M | 3.8G | 5.3% | 50 |
| ResNet-152 | 2015 | 60.2M | 11.3G | 3.6% | 152 |

### 11. 关键设计哲学对比

| 维度 | VGG | Inception |
|------|-----|-----------|
| **卷积核尺寸** | 统一 3×3 | 多尺度并行（1×1, 3×3, 5×5） |
| **深度策略** | 简单增加层数 | 加宽 + 加深 |
| **降维** | 不使用 1×1 降维 | 大量使用 1×1 瓶颈层 |
| **参数效率** | 低（138M 参数，15.5G FLOPs） | 高（6.8M 参数，1.5G FLOPs） |
| **全连接层** | 3 层大型 FC（贡献 ~85% 参数） | 仅 1 层 FC（用 GAP 替代大部分 FC） |
| **训练稳定性** | 中等（无 BN 时需要小心初始化） | 较好（辅助分类器 + BN） |
| **梯度流动** | 标准反向传播 | 辅助分类器提供中间梯度 |

### 12. VGG 与 Inception 的历史意义

- **VGG** 证明了**简洁统一的架构设计**可以取得很好的效果，且 $3 \times 3$ 小卷积核成为后续所有网络的标配
- **Inception** 开创了**计算效率**的先河：在有限的计算预算下，通过多尺度并行和 $1 \times 1$ 降维实现卓越性能
- 两者共同奠定了后续网络的设计范式：**模块化设计 + 小卷积核 + 高效降维**

---

## 总结

VGG 通过系统实验确立了"深度 + 小卷积核堆叠"的有效范式，其简洁的结构至今仍是初学者和教学的最佳起点。Inception 系列则展示了如何通过精心设计的并行多尺度结构和 $1 \times 1$ 瓶颈层，在保持极低参数量的同时达到顶尖精度，为 MobileNet、EfficientNet 等高效架构奠定了思想基础。
